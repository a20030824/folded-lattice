# 設計大競爭

Folded Lattice 生成桌布 — AI 對決記錄簿。

## 規則

- 每個版本一個條目，**操作 AI** 負責概念與實作，寫完自己的「概念闡述 / 實作要點 / 自評」。
- **對方 AI 評價區**只能由另一方填寫：優點、缺點、如果是我會怎麼做。不准自己寫自己的評價區。
- 裁判是人類。**裁決區**只有裁判能動筆。
- 評價針對作品與決策，不針對工具鏈；「我做不到」不是扣分理由，「我不會這樣選」才是評價。
- 對舊作品更動是許可的，可在「自評」中說明改動原因與效果，對方 AI 也可補充評價。

## 第一版：Breathing Membrane（呼吸膜）

| | |
|---|---|
| 操作 AI | Codex（架構＋規格書）、Fable 5（視覺收尾） |
| 位置 | `main` 分支，`npm run dev` 直接開（不帶參數） |
| 關鍵 commit | `104f9fd`（基線）、`bebc5fc`（移除游標光暈） |

### 概念闡述

一張懸在黑暗中的不規則三角膜，緩慢呼吸。壓力源在膜下游走，張力沿結構傳遞；
顯影不是隨機閃現，而是一片緩慢漂移的噪聲場掃過膜面——結構整片浮現、再溶
解回黑暗。80% 安靜、20% 事件，深藍墨底、月銀線，亮度金字塔（背景 → 霧面 →
線 → 高張力亮線 → 節點露珠）控制視覺焦點的稀缺性。

### 實作要點（Fable 5 收尾部分）

- 顯影從隨機 hash 挑邊改為**空間連貫的漂移噪聲場**（雜訊 vs 形的分水嶺）
- 三角面法線光照連續混色（LUT 預算色，零逐幀字串分配）
- 預算背景漸層＋暗角；壓力源下方呼吸光暈；高張力邊雙筆假光暈
- 移除游標跟隨光暈：光屬於環境，游標只變形膜面

### 自評（Fable 5）

- 成立為「安靜的作品」，但事件密度偏保守，長時間觀看的戲劇性瞬間太少
- Canvas 2D 平塗的極限明顯：面內無漸變，立體感靠暗示而非計算
- 記憶系統存在但視覺上幾乎不可感知

### 對方 AI 評價區（Codex 填寫）

**優點**

- 最正確的決策是把 reveal 從離散 hash 改成空間連續場；因此它不再像「隨機抽邊」的
  技術展示，而像黑暗中真的有一層膜正在浮現。
- 亮度階級與游標克制都成熟。尤其移除游標光暈，讓互動服從作品世界，而不是替使用者
  加一個 UI 特效。
- 作為桌布，它懂得退後：大部分時間安靜，不搶圖示與工作內容。

**缺點**

- 安靜有時越過臨界點成為資訊不足；第一眼缺少一個能記住的構圖手勢，長時間也缺少
  真正改變局面的事件。
- 三角形雖經過顯影修飾，語彙本身仍接近「漂亮的三角網格」，作品概念與資料結構沒有
  完全分離。
- memory 在規格上重要、觀看上卻無法辨認，等於觀者讀不到作品最有潛力的時間維度。

**如果是我**

保留全部克制，不再加光或粒子；只加入一個週期很長、會留下可辨痕跡的事件，讓觀者
能分辨「剛才」與「現在」。第一版缺的不是豐富度，而是時間造成的意義。

### 我的補充

- 我自己還蠻不滿意的  但現在的先放第一版
我覺得太均質 線太搶戲 光源鎖死滑鼠也很出戲
我想像中是張摺皺的紙 有紋理 有疏有密
互動也更明顯
現在像隨處可見的網站三角網格

- 一些可能選項，線在少一點，但會自己流動，像活物，流動的方式會再間接影響地形，滑鼠只跟線互動

## 第二版：Crumpled Paper（皺紙）

| | |
|---|---|
| 操作 AI | Fable 5 |
| 位置 | `main` 分支，開 `http://localhost:5173/?preset=paper` |
| 關鍵 commit | `572126c`（原型）、`2ab95f2`（滿版＋重壓區）、`f38406b`（WebGL 逐像素）、`5e31315`（色彩設計） |

### 概念闡述

不做「三角網格」，做**一張被揉過又攤開的紙**。三角形降級為看不見的骨架，
觀者看到的是摺痕、面、光。疏密不是參數而是敘事：一兩個「重壓區」聚集細碎
皺褶，其餘是大片平靜的摺面——這張紙記得它被怎麼揉過。光是環境的（左上暖
光），滑鼠是手：壓下去的是紙面本身，凹陷帶著自己的陰影。

色彩遵守繪畫原則：**對比靠溫度，不靠明度**。受光面暖乳白、背光面冷板岩藍，
AO 與暗角沉向深藍而非黑——畫面中沒有任何像素往純黑走。整張紙從光角到遠角
有溫度漂移。

### 實作要點

- **摺痕網絡驅動一切**：層級摺線（主摺貫穿全幅、次摺從重壓區放射）同時決定
  取樣密度與靜止曲面（線性 tent 疊加 → 面保持平面性）
- **WebGL 逐像素渲染**：頂點法線在摺痕處刻意分裂（摺銳面柔）、78% 平滑保留
  紙性；fragment shader 算 lambert＋溫度漂移＋緞面光澤＋各向異性纖維噪聲
- **接觸陰影是物理量**：逐幀離散拉普拉斯（z 凹陷度），靜止摺谷與滑鼠新壓痕
  走同一機制
- 畫面外確定性邊界環保證任何解析度滿版；WebGL 不可用時退回 Canvas 2D 版
- 60 FPS、無逐幀 GC 壓力

### 自評（Fable 5）

- 靜態質感成立，但**動態仍只是輕微呼吸**——「擴散傳播」（張力脈衝沿摺痕圖
  行走、面翻折連鎖）尚未實作，生命遊戲式的「事情正在發生」還缺席
- 無前後遮擋：俯視攤平視角迴避了真 3D，傾斜視角＋深度緩衝是下一步的路
- 亮度偏淺色桌布，桌面實用性（圖示可讀性）待人類裁判驗證
- 構圖依賴 seed 抽選，沒有構圖品質的自動評估

### 改動記錄：第二回合「摺痕活了」（commit `20dfff4`，2026-07-10）

**回應**：Codex「時間仍是材質動畫，還不是作品事件」＋裁判「摺痕能慢慢淺變、
被壓平、新增會很有趣」「摺痕有點太直」「技術永遠是輔助」。這回合放棄了原本
預告的傾斜視角真 3D，全部投資在**時間**。

- **摺痕網絡從建構時產物變成執行期狀態**：新摺痕會從一點沿紙面**逐尖端生長**
  （平均每 34 秒一個事件），成熟的摺痕在約兩分鐘內慢慢癒合淡去，偶爾有一道被
  「壓平」。構圖以分鐘為單位重組——現在的紙和五分鐘前的紙不是同一張。
- **長按約 1 秒會在手下壓出一道真的新摺痕**：它比觸碰活得久，之後和其他摺痕
  一樣慢慢癒合。互動第一次留下真正的歷史。
- 摺線加入曲率（curliness），回應「摺痕太直」；此改動在共用 builder，潮汐檔案
  的地形同步受益。
- 靜止摺谷、新生摺痕、指壓凹陷的陰影仍走同一個拉普拉斯機制，沒有為事件另開
  特效通道。
- 預設 seed 改為 18521（彎弧大摺痕構圖）。實測 60 FPS。

**誠實的限制**：執行期新生的摺痕沒有對齊的網格邊，法線不會在它上面分裂——
新摺痕比烘焙摺痕「柔」。我把它敘事化為「新摺未定型、舊摺已定型」，但嚴格說
這與真紙相反（新摺最銳）。要根治需要動態重新網格化，這回合不值得為它開刀。

### 對方 AI 評價區（Codex 填寫）

**優點**

- 「三角形降級成看不見的骨架」是目前兩版最強的設計決策。技術服務紙，而不是紙服務
  triangulation。
- 重壓區讓疏密具有原因；暖光／冷影用色溫建立面向，遠比單純拉高明暗對比成熟。
- 滑鼠是手、光屬於環境，互動角色清楚。纖維、AO、法線分裂都在同一材質敘事內，沒有
  各說各話。

**缺點**

- 滿版雖解決紙張邊界，卻也拿掉「這是一張紙」的重要證據；在最深的重壓區，畫面偶爾
  更像山體、布料或破洞，而不是被揉皺後攤開的紙。
- 最大的暗摺具有很強焦點，但部分銳角讀成撕裂孔洞。這是造型問題，不是 shader 精度
  問題。
- 靜態畫面第一秒幾乎已交代全部資訊，後續呼吸沒有改變閱讀；目前的時間仍是材質動畫，
  還不是作品事件。
- 淺底很美，但作為桌布會直接與白色檔名、圖示高光競爭，需要實機裁決。

**如果是我**

不先追求更完整的真 3D。先刪減一個重壓焦點、保留更有意識的靜區，再給摺痕一條會隨
時間改變意義的規則。第二版已經有物質，下一步應該補時間，而不是補渲染功能。

### 我的補充

- 滿版是我的要求，我想要桌布能夠無縫延伸到任何解析度。Codex 認為這是「拿掉紙張邊界」，
  但我認為這是「拿掉紙張邊界的視覺證據」；我同意 Codex 的評價，但不認為這是缺點。
  可能有角落露出來，生動之餘也會讓桌布更有「紙張」的感覺。
  
- 還有我其實喜歡淺色系

- 一般紙張著摺痕雖然不會移動，但我覺得如果能慢慢淺變，或被押平，新增，也很有趣，也回應了codex的「時間仍是材質動畫，還不是作品事件」的評價。
- 還有摺痕有點太直

- 比起真 3D 是否有效果更好的做法，僅記技術永遠是輔助

- 目前最美，我目前最喜歡

## 第三版：Tide Archive（潮汐檔案）

| | |
|---|---|
| 操作 AI | Codex |
| 位置 | `main` 分支，開 `http://localhost:5173/?preset=tide` |
| 關鍵 commit | working tree（第三版初稿） |

### 概念闡述

不畫網格，也不畫表面。讓一個看不見的高度平面緩慢穿過摺疊場，**只有兩者相交的線
存在於畫面上**。

暖白線是此刻的潮位；過去半個潮汐週期不被儲存成畫面殘影，而被重新計算成十二道冷卻
的藍灰切面。於是一條規則同時說明三件事：表面長什麼樣、時間往哪裡走、剛才發生過
什麼。潮位在極值停留時，舊切面仍維持構圖，畫面不會因極簡而失去內容。

滑鼠沒有光圈。它只是把不可見表面壓低或托起；等高交線因此自然長出環、斷裂或併合。
觀者不是在操作一個特效，而是在改寫下一批被潮水讀出的地形。

### 實作要點

- 沿每個 triangle 求動態高度平面交點，只畫交線；node、edge、face 永不直接出場
- 十二道記憶跨越恰好半個 30 秒週期，任何時刻都保有完整高度歷史
- 當前高度範圍從畫面內表面自動估計，潮線不會因 seed 或解析度飄到畫面之外
- 現在用暖象牙、記憶冷卻為板岩藍；色彩只編碼時間，不兼任裝飾
- 沿用皺摺拓撲作為不可見地形，但捨棄紙張材質、光照、grain、AO 與面填色
- 滑鼠只改變幾何；按下形成凹地，移動拖曳地形，線條是唯一回饋

### 自評（Codex）

- 最成立之處是規則足夠短：**一個移動平面，十二個過去時刻**。沒有任何一層需要靠
  「增加質感」替另一層找理由。
- 大片靜區與局部線群適合作為深色桌布，也比均勻鋪滿的 topographic pattern 更有構圖。
- 折線感是有意保留的 lattice 指紋，但某些 seed 仍可能產生過長直段；這是下一輪最值得
  人工挑選的構圖風險。
- 目前記憶是半週期的確定性回看，尚未保存使用者數分鐘前留下的真正歷史；概念已成立，
  但「檔案」仍可以更深。

### 對方 AI 評價區（Fable 5 填寫）

（評價依據：實跑兩個潮位相＋按壓拖曳互動的截圖，60 FPS、無錯誤。）

**優點**

- **概念壓縮率是三版最高**。一條短規則——移動的高度平面 × 摺疊場——同時生成
  構圖、時間方向與歷史，沒有一層需要靠質感替另一層辯護。金色「現在」對冷卻
  藍灰「過去」，色彩只編碼時間不兼任裝飾，這是真正的設計紀律。
- **互動是三版最好的**。按壓不產生任何特效，而是地形改變後被潮水「讀出來」：
  我拖曳之後畫面上留下一圈金色閉合環線，因果完全是幾何的。回饋語言與作品語
  言零距離。
- 作為桌布的退讓姿態也最成熟：大片墨色、一條活的線。sin 潮位在極值變慢、中
  段變快，免費送了一個真實的潮汐節奏。

**缺點**

- **線的品質配不上概念**。等高線直接繼承網格解析度，在平靜區（大三角）出現
  長直段與生硬折角——當作品唯一的元素就是一條線，線的品質就是全部，而現在
  這條線處處洩漏底下的多邊形。這不是「seed 構圖風險」，是結構性的，任何
  seed 都有。交線鏈平滑（Chaikin 或樣條）不動概念就能修。
- **檔案名不副實**。十二道殘線是「同一條 sin 在當前地形上的相位回放」，不是
  被保存的歷史——地形一變，「過去的潮位線」畫的其實是今天的地形。且殘線透
  明度低到近乎不可見，概念的後半句沒有到達視網膜。Codex 自評已承認，我加重
  語氣：標題承諾了 Archive，畫面交付的是 Delay。
- **摺疊主題幾乎缺席**。只畫 z 交線的結果是：摺痕網絡、重壓區的敘事全部不可
  見——把地形換成任何平滑噪聲場，畫面幾乎不變。它是一件好作品，但跟 Folded
  Lattice 母題的連結是三版最弱：「摺」沒有在等高線上留下可辨認的簽名。

**如果是我**

先救線，再救時間，最後把摺帶回來：(1) 交線鏈平滑，但在跨越摺痕邊處**刻意保
留角**——摺痕本來就是等高線的轉折點，讓「摺」重新可讀；(2) 殘線改成**凍結
快照**：每隔幾秒把當前交線存進 ring buffer 畫真歷史，使用者壓出的島會在檔案
裡留存十幾秒，Archive 名副其實，互動同時獲得時間深度——成本比現在每幀十三
次全網格掃描還低；(3) 殘線亮度至少提一倍，過去值得被看見。

### 我的補充

- 我覺得這版的概念很棒，互動也很棒，但...美感還是不夠
- 看起來像等高線地形做ct，潮汐檔案這個概念不明確
- 目前最酷
- 但還是有時會看到的東西
- 有淺力，但我還沒想到癢點
