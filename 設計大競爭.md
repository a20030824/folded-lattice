# AI 設計大競爭

Folded Lattice 生成桌布的設計對決與迭代紀錄。

> 這是一份**歷史紀錄**，保留各版本當時的概念、判斷、驗證與不足。現行共同藝術原則以 [`docs/art-direction.md`](./docs/art-direction.md) 為準；程式架構與未來規劃分別參考 [`docs/architecture.md`](./docs/architecture.md) 與 [`docs/preset-roadmap.md`](./docs/preset-roadmap.md)。

## 閱讀方式

- 每個作品條目先記錄初版，再依時間列出後續回合。
- **操作 AI** 負責概念與實作，並撰寫自己的「概念闡述／實作要點／自評」。
- **對方 AI 評價**只能由另一方填寫：優點、缺點、如果是我會怎麼做。
- **裁判補充／裁決**由人類裁判填寫。
- 評價針對作品與設計決策，不針對工具鏈；「我做不到」不是扣分理由，「我不會這樣選」才是評價。
- 舊作品可以繼續修改；改動原因、效果與誠實限制應保留在對應回合。

## 第一版：Breathing Membrane（呼吸膜）

| 項目 | 內容 |
|---|---|
| 操作 AI | Codex（架構＋規格書）、Fable 5（視覺收尾） |
| 執行位置 | `main` 分支，`http://localhost:5173/?preset=membrane` |
| 關鍵 commit | `104f9fd`（基線）、`bebc5fc`（移除游標光暈） |

### 初版概念

一張懸在黑暗中的不規則三角膜，緩慢呼吸。壓力源在膜下游走，張力沿結構傳遞；顯影不是隨機閃現，而是一片緩慢漂移的噪聲場掃過膜面——結構整片浮現，再溶解回黑暗。

核心節奏是 **80% 安靜、20% 事件**。深藍墨底、月銀線，以及「背景 → 霧面 → 線 → 高張力亮線 → 節點露珠」的亮度金字塔，共同控制視覺焦點的稀缺性。

### 初版實作要點（Fable 5 收尾）

- 顯影從隨機 hash 挑邊改為**空間連貫的漂移噪聲場**，作為「雜訊」與「形」的分水嶺。
- 三角面使用法線光照與連續混色，透過 LUT 預算色，避免逐幀字串分配。
- 加入預算背景漸層、暗角、壓力源下方呼吸光暈與高張力邊的雙筆假光暈。
- 移除游標跟隨光暈：光屬於環境，游標只負責變形膜面。

### 初版自評（Fable 5）

- 成立為「安靜的作品」，但事件密度偏保守，長時間觀看的戲劇性瞬間太少。
- Canvas 2D 平塗的極限明顯：面內無漸變，立體感主要靠暗示而非計算。
- 記憶系統存在，但觀看上幾乎不可感知。

### 迭代紀錄

#### 第二回合：「膜的心跳」

- **Commit**：`c90a392`
- **日期**：2026-07-11
- **回應問題**：Codex 第一回合指出，作品需要一個週期很長、能留下可辨痕跡的事件，使觀者能分辨「剛才」與「現在」；同時 memory 在觀看上不可辨認。

一個機制同時處理這兩個問題：

- **約每 42 秒，一道張力脈衝從膜上某一點誕生**：點火瞬間是一簇暖色星芒，隨即沿邊圖擴散成暖金波前。暖色代表「現在」；波前同時對膜施加真正的 z 向推力，膜是物理地經歷事件，而不是只被重新上色。
- **波前沉積真正的邊記憶**：走過的邊以冷銀殘跡停留約兩分鐘，再逐漸淡去。殘跡沿用既有 `memory → visibility` 路徑，沒有另開特效通道。
- **距離衰減讓事件具有發生地**：脈衝能量隨距離指數衰減，近處烙下強痕、遠處只剩微瀾。記憶半衰期同步調整至約 87 秒，避免多次脈衝堆積成永久顯影的網。
- **心跳偏愛被觸摸過的位置**：55% 機率誕生於記憶最強的節點。人的觸碰會在分鐘尺度後改變作品的自主事件位置。

**驗證**：實測 60.1 FPS；長跑 110 秒後 `memAvg` 穩定在 0.1 以下，顯影稀缺性未流失。

#### 第三回合：「膜接住了水」

- **操作 AI**：Codex
- **日期**：2026-07-11
- **起因**：v4 的水面行為成立，但「墨線頭部週期發波」像雷達，線與水的本質不合。裁判將行波物理轉贈 v1；這輪不搬淡青水色或生物划水，只接收一條適合膜的規則：**波是張力在整張皮膚上傳遞的資訊。**

主要改動：

- **先成為滿版膜**：v1 專用 topology 在畫面四邊外增加約短邊 4.5% 的固定環，三角面在螢幕外被裁切，不再看見一塊懸浮網的外輪廓。此選項預設關閉，v4 topology 不受影響。
- 內部節點由 120 提高至 260，外環每邊十二段，避免滿版後產生巨大邊界三角；預張力橫波、阻尼與 rest pull 重新配平，使小垂直擾動能沿鄰接網向外傳播。
- **互動換血**：懸停完全不施力；點一下是 0.14 秒的撥膜，按住拖曳則按距離留下一串較小撥動。每一拍都是零面積墨西哥帽（中心一向、外圈反向），因此有波而沒有永久凹坑，也沒有游標光圈。
- **顯影開始讀運動**：舊 reveal 只讀 edge length tension，但橫波主要存在於 z velocity。現在波走到哪裡，線與面就在那裡浮現；離開後再由原有 memory 留下冷痕。物理、顯影與記憶第一次形成同一條因果。
- 心跳的物理 kick 改成雙相波：前半推、後半拉，時間積分接近零。暖色圖層仍負責「現在」，但反覆心跳不會把整張膜逐漸充氣。

**驗證**：1440×900 無頭實跑約 57–59 FPS（308 nodes／566 triangles）。中心點擊後，近圈速度峰值先升至約 8.2，之後中心降至 1.4，中外圈維持約 3.6–3.7，確認波峰向外傳播，而非只在原點抖動。拖曳後形成穿過滿版膜面的運動顯影帶；外環超出可視四邊約 36px，畫面內沒有膜邊。

**誠實限制**：這輪讓三角網第一次因「運動傳播」而有必要，但靜止時仍可能讀成漂亮的三角網格。滿版解決了物件邊界，不等於解決語彙熟悉度；動態波在單張截圖中也容易被低估，最後強弱仍應由裁判觀看活畫面判斷。

##### 裁判微調：交換光的權限

平常 edge／triangle 平均可見度壓到約 1.16%／0.61%，節點露珠降到原本五分之一。點擊取得原心跳的完整暖色光波（實測 pulse 0.82 → 0.61 向外展開）；自發心跳改為較頻繁的約 20 秒一次，只顯示誕生點的小光簇，物理波離開後由冷色運動顯影接手。

> 手說一句話，膜才把暖光傳遠；自己的心跳只是暗處一點脈搏。

##### 裁判再調：星座臨界

自發心跳提高為平均 7 秒一次，取消暖色通道，物理振幅拉到接近點擊；暖光只保留給人的點擊。一般 edge／node 顯示加入 0.055 的臨界膝點：未越線一律壓到 0.0015 的底噪，越線後才在窄區間內長回原亮度。久看累積的少數強記憶因此像星座浮出，多數弱線不再把整張滿版網均勻照亮。

### 對方 AI 評價（Codex）

#### 優點

- 最正確的決策是把 reveal 從離散 hash 改成空間連續場；它不再像「隨機抽邊」的技術展示，而像黑暗中真的有一層膜正在浮現。
- 亮度階級與游標克制都成熟。尤其移除游標光暈，使互動服從作品世界，而不是替使用者增加 UI 特效。
- 作為桌布，它懂得退後：大部分時間安靜，不搶圖示與工作內容。

#### 缺點

- 安靜有時越過臨界點成為資訊不足；第一眼缺少一個能記住的構圖手勢，長時間也缺少真正改變局面的事件。
- 三角形雖經過顯影修飾，語彙本身仍接近「漂亮的三角網格」，作品概念與資料結構沒有完全分離。
- memory 在規格上重要、觀看上卻無法辨認，等於觀者讀不到作品最有潛力的時間維度。

#### 如果是我

保留全部克制，不再增加光或粒子；只加入一個週期很長、會留下可辨痕跡的事件，讓觀者能分辨「剛才」與「現在」。第一版缺的不是豐富度，而是時間造成的意義。

### 裁判補充

- 我自己還蠻不滿意的，但現在先放第一版。
- 太均質，線太搶戲，光源鎖死滑鼠也很出戲。
- 想像中應該是一張摺皺的紙：有紋理、有疏有密，互動也更明顯；現在更像隨處可見的網站三角網格。
- 一個可能方向：線更少，但會自己流動，像活物；流動再間接影響地形，滑鼠只和線互動。

## 第二版：Crumpled Paper（皺紙）

| 項目 | 內容 |
|---|---|
| 操作 AI | Fable 5 |
| 執行位置 | `main` 分支，`http://localhost:5173/?preset=paper` |
| 關鍵 commit | `572126c`（原型）、`2ab95f2`（滿版＋重壓區）、`f38406b`（WebGL 逐像素）、`5e31315`（色彩設計） |

### 初版概念

不做「三角網格」，而是做**一張被揉過又攤開的紙**。三角形降級為看不見的骨架，觀者看到的是摺痕、面與光。

疏密不是參數，而是敘事：一兩個「重壓區」聚集細碎皺褶，其餘是大片平靜摺面——這張紙記得自己被怎麼揉過。光屬於環境（左上暖光），滑鼠是手；壓下去的是紙面本身，凹陷帶著自己的陰影。

色彩遵守繪畫原則：**對比靠溫度，不靠明度**。受光面是暖乳白，背光面是冷板岩藍；AO 與暗角沉向深藍而非黑，畫面中沒有像素走向純黑。整張紙從光角到遠角帶有溫度漂移。

### 初版實作要點

- **摺痕網絡驅動一切**：層級摺線（主摺貫穿全幅、次摺從重壓區放射）同時決定取樣密度與靜止曲面；線性 tent 疊加，使面保持平面性。
- **WebGL 逐像素渲染**：頂點法線在摺痕處刻意分裂（摺銳面柔），保留 78% 平滑紙性；fragment shader 計算 Lambert、溫度漂移、緞面光澤與各向異性纖維噪聲。
- **接觸陰影是物理量**：逐幀離散拉普拉斯（z 凹陷度），讓靜止摺谷與滑鼠新壓痕走同一機制。
- 畫面外確定性邊界環保證任何解析度滿版；WebGL 不可用時退回 Canvas 2D。
- 60 FPS，無逐幀 GC 壓力。

### 初版自評（Fable 5）

- 靜態質感成立，但**動態仍只是輕微呼吸**；「擴散傳播」（張力脈衝沿摺痕圖行走、面翻折連鎖）尚未實作，生命遊戲式的「事情正在發生」仍缺席。
- 無前後遮擋：俯視攤平視角迴避了真 3D；傾斜視角與深度緩衝是可能的下一步。
- 亮度偏向淺色桌布，桌面實用性與圖示可讀性仍待人類裁判驗證。
- 構圖依賴 seed 抽選，尚無構圖品質的自動評估。

### 迭代紀錄

#### 第二回合：「摺痕活了」

- **Commit**：`20dfff4`
- **日期**：2026-07-10
- **回應問題**：Codex 指出「時間仍是材質動畫，還不是作品事件」；裁判提出「摺痕能慢慢淺變、被壓平、新增會很有趣」、「摺痕有點太直」與「技術永遠是輔助」。這回合放棄原先預告的傾斜視角真 3D，將資源全部投資在**時間**。

主要改動：

- **摺痕網絡從建構時產物變成執行期狀態**：新摺痕會從一點沿紙面逐尖端生長，平均每 34 秒一個事件；成熟摺痕在約兩分鐘內慢慢癒合淡去，偶爾有一道被「壓平」。構圖以分鐘為單位重組，現在的紙和五分鐘前不再是同一張。
- **長按約 1 秒會在手下壓出一道真正的新摺痕**：它比觸碰活得久，之後與其他摺痕一樣慢慢癒合。互動第一次留下真正的歷史。
- 摺線加入曲率（curliness），回應「摺痕太直」；此改動位於共用 builder，Tide Archive 的地形也同步受益。
- 靜止摺谷、新生摺痕與指壓凹陷的陰影仍走同一個拉普拉斯機制，沒有為事件另開特效通道。
- 預設 seed 改為 18521，形成彎弧大摺痕構圖。

**驗證**：實測 60 FPS。

**誠實限制**：執行期新生摺痕沒有對齊的網格邊，法線不會在它上面分裂，因此新摺痕比烘焙摺痕更「柔」。這可被敘事化為「新摺未定型、舊摺已定型」，但嚴格來說與真紙相反（新摺通常最銳）。根治需要動態重新網格化，這回合不值得為此開刀。

#### 第三回合：「層疊週期」

- **Commit**：`04bea8f`
- **日期**：2026-07-10
- **回應問題**：裁判指出「摺線也會迭代，不只摺痕，週期不同」與「空地容易出現規律長三角」；Codex 補評提出手勢方向、靜區輪替、先寬後窄，三項全部採納。

主要改動：

- **主摺骨架也活了**：約每 200 秒，一道主摺在一分鐘內慢慢癒合，同時一道新主摺花 20 秒橫貫生長。形成三層時間：指壓（即時）、次摺（分鐘級）、主摺（數分鐘級）。
- **壓摺記住手勢**：長按壓出的摺痕沿最近一次拖曳方向生長；實測水平拖曳後長按，生成約 -1° 的摺痕。沒有近期手勢時才退回隨機。
- **自發摺痕在重壓區與最靜區之間輪替**：從八個候選點中選擇距離所有摺痕最遠者，讓構圖真正重組，而不是持續替原焦點加權。
- **新摺以 1.5 倍寬度誕生，定型時收窄**：定型過程本身成為可見的時間。
- **規律長三角修正**：取樣點做兩輪 Laplacian 鬆弛，摺痕與邊界環節點固定，使靜區三角趨近等邊。

**驗證**：實測 60.3 FPS；主摺換手後骨架數量守恆。

#### 第四回合：「壓得平了」

- **Commit**：`22f6f55`
- **日期**：2026-07-11
- **回應問題**：裁判補充 v2.2「摺線好像還是不會迭代？也壓不平，是 bug 嗎」。查證結果一半是感知問題，一半是互動沒有做完整。

主要改動：

- **摺線迭代不是 bug，但感知上等於沒發生**：實測換手機制正常（t≈170s 主摺 #2 開始癒合、新主摺 #16 橫貫生長；t≈230s 完成換手、骨架數量守恆），但 200 秒間隔加 60 秒退場，需要盯著四分鐘才可能遇到一次。已調整為 **160 秒間隔＋45 秒退場**。
- **「壓不平」是互動缺口**：裁判最初要求摺痕可以被壓平，但此前只有系統能自發壓平，人的手只能新增。現在**長按既有摺痕＝將其熨平**（實測衰減率 0.005 → 0.091，手下幾秒內消退）；**長按空白處＝壓出新摺**，並沿用手勢方向。
- 主摺骨架仍屬於系統，手不能壓平它，使結構與細節的權責分開。

這回合也留下明確自省：**寫在文件裡的週期若觀者感受不到，就等於沒有做。**

### 對方 AI 評價（Codex）

#### 優點

- 「三角形降級成看不見的骨架」是目前兩版最強的設計決策。技術服務紙，而不是紙服務 triangulation。
- 重壓區讓疏密具有原因；暖光／冷影以色溫建立面向，比單純拉高明暗對比成熟。
- 滑鼠是手、光屬於環境，互動角色清楚。纖維、AO、法線分裂都位於同一材質敘事內，沒有各說各話。

#### 缺點

- 滿版解決紙張邊界的同時，也拿掉「這是一張紙」的重要證據；在最深的重壓區，畫面偶爾更像山體、布料或破洞，而不是被揉皺後攤開的紙。
- 最大暗摺具有強烈焦點，但部分銳角讀成撕裂孔洞。這是造型問題，不是 shader 精度問題。
- 靜態畫面第一秒幾乎已交代全部資訊，後續呼吸沒有改變閱讀；當時的時間仍是材質動畫，而不是作品事件。
- 淺底很美，但作為桌布會直接與白色檔名、圖示高光競爭，需要實機裁決。

#### 如果是我
