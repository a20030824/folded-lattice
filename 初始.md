下面先定成 **Folded Lattice Engine v0.1**。核心原則是：

* `Config`：不隨動畫改變的作品設定
* `State`：每幀都會變化的模擬狀態
* `System`：對 State 進行更新的模組
* `Preset`：把拓撲、物理、顯影、渲染參數組合成作品
* 節點、邊、三角形都用數字索引互相引用，避免大量物件查找
* V1 使用 Canvas 2D，但模擬狀態保留 `z`，製造摺疊與光影

# 3. `core/state.ts`

## Node

節點保留三維位置：

* `x/y`：畫面中的平面位置
* `z`：虛擬高度，用於凹凸與光影
* `restPosition`：最初平衡位置
* `memoryOffset`：歷史形變造成的微小偏向

```ts
// src/core/state.ts

import type {
  PointerState,
  TimeState,
  Vec3,
  Viewport,
} from "./types";

export interface NodeState {
  id: number;

  position: Vec3;
  previousPosition: Vec3;
  velocity: Vec3;
  force: Vec3;

  /**
   * 初始且永久的基準位置。
   */
  restPosition: Vec3;

  /**
   * 記憶系統造成的暫時平衡偏移。
   * 真正的目標位置可視為 restPosition + memoryOffset。
   */
  memoryOffset: Vec3;

  mass: number;
  inverseMass: number;

  /**
   * 是否固定。
   * 邊界可以固定，或只降低其自由度。
   */
  pinned: boolean;

  /**
   * 與此節點相連的 edge index。
   */
  edgeIndices: number[];

  /**
   * 包含此節點的 triangle index。
   */
  triangleIndices: number[];
}
```

## Edge

`strain` 和 `tension` 分開：

* `strain`：相對形變比例，可正可負
* `tension`：經平滑後的視覺與記憶用強度

```ts
export interface EdgeState {
  id: number;

  nodeA: number;
  nodeB: number;

  /**
   * 原始長度。
   */
  baseRestLength: number;

  /**
   * 目前受記憶影響後的平衡長度。
   */
  restLength: number;

  currentLength: number;

  /**
   * (currentLength - restLength) / restLength
   */
  strain: number;

  /**
   * 視覺化使用的平滑張力，通常取 abs(strain) 再平滑。
   */
  tension: number;

  /**
   * 長期受力累積。
   * 建議維持在 0～1。
   */
  memory: number;

  /**
   * 顯影系統輸出的平滑可見度。
   */
  visibility: number;

  /**
   * 事件性高亮。
   */
  highlight: number;
}
```

# 7. System 的實作形式

每一個系統都保持同一格式。

例如 spring system：

```ts
// src/core/simulation/applySprings.ts

import type { SimulationSystem } from "../contracts";

export const springSystem: SimulationSystem = {
  name: "spring-system",

  update(state, config, deltaSeconds) {
    const { nodes, edges } = state.topology;
    const physics = config.physics;

    // 每次 fixed update 開頭清空 force。
    for (const node of nodes) {
      node.force.x = 0;
      node.force.y = 0;
      node.force.z = 0;
    }

    for (const edge of edges) {
      const a = nodes[edge.nodeA];
      const b = nodes[edge.nodeB];

      const dx = b.position.x - a.position.x;
      const dy = b.position.y - a.position.y;
      const dz = b.position.z - a.position.z;

      const length = Math.hypot(dx, dy, dz);

      if (length < 1e-6) {
        continue;
      }

      const extension = length - edge.restLength;
      const normalizedX = dx / length;
      const normalizedY = dy / length;
      const normalizedZ = dz / length;

      const planarForce =
        extension *
        physics.springStrength *
        physics.planarSpringStrength;

      const verticalForce =
        extension *
        physics.springStrength *
        physics.verticalSpringStrength;

      if (!a.pinned) {
        a.force.x += normalizedX * planarForce;
        a.force.y += normalizedY * planarForce;
        a.force.z += normalizedZ * verticalForce;
      }

      if (!b.pinned) {
        b.force.x -= normalizedX * planarForce;
        b.force.y -= normalizedY * planarForce;
        b.force.z -= normalizedZ * verticalForce;
      }
    }

    // 弱力拉回 rest pose。
    for (const node of nodes) {
      if (node.pinned) {
        continue;
      }

      const targetX =
        node.restPosition.x + node.memoryOffset.x;
      const targetY =
        node.restPosition.y + node.memoryOffset.y;
      const targetZ =
        node.restPosition.z + node.memoryOffset.z;

      node.force.x +=
        (targetX - node.position.x) *
        physics.restPoseStrength;

      node.force.y +=
        (targetY - node.position.y) *
        physics.restPoseStrength;

      node.force.z +=
        (targetZ - node.position.z) *
        physics.restPoseStrength;
    }

    void deltaSeconds;
  },
};
```

這裡有一個重要規則：

> 只有負責積分的 system 可以直接改 `position` 和 `velocity`；其他 force system 只累加 `force`。

這能避免不同模組互相打架。

# 8. Integration system

```ts
// src/core/simulation/integrate.ts

import type { SimulationSystem } from "../contracts";

function clampMagnitude(
  x: number,
  y: number,
  z: number,
  maximum: number,
): [number, number, number] {
  const magnitude = Math.hypot(x, y, z);

  if (magnitude <= maximum || magnitude < 1e-6) {
    return [x, y, z];
  }

  const scale = maximum / magnitude;

  return [
    x * scale,
    y * scale,
    z * scale,
  ];
}

export const integrationSystem: SimulationSystem = {
  name: "integration-system",

  update(state, config, deltaSeconds) {
    const { nodes } = state.topology;
    const {
      damping,
      maximumVelocity,
      maximumDepthRatio,
    } = config.physics;

    const shortSide = Math.min(
      state.viewport.width,
      state.viewport.height,
    );

    const maximumDepth =
      shortSide * maximumDepthRatio;

    /**
     * 指數阻尼比 velocity *= 0.95 更能適應不同 dt。
     */
    const dampingFactor =
      Math.exp(-damping * deltaSeconds);

    for (const node of nodes) {
      node.previousPosition.x = node.position.x;
      node.previousPosition.y = node.position.y;
      node.previousPosition.z = node.position.z;

      if (node.pinned) {
        node.velocity.x = 0;
        node.velocity.y = 0;
        node.velocity.z = 0;
        continue;
      }

      node.velocity.x +=
        node.force.x *
        node.inverseMass *
        deltaSeconds;

      node.velocity.y +=
        node.force.y *
        node.inverseMass *
        deltaSeconds;

      node.velocity.z +=
        node.force.z *
        node.inverseMass *
        deltaSeconds;

      node.velocity.x *= dampingFactor;
      node.velocity.y *= dampingFactor;
      node.velocity.z *= dampingFactor;

      const [vx, vy, vz] = clampMagnitude(
        node.velocity.x,
        node.velocity.y,
        node.velocity.z,
        maximumVelocity,
      );

      node.velocity.x = vx;
      node.velocity.y = vy;
      node.velocity.z = vz;

      node.position.x += vx * deltaSeconds;
      node.position.y += vy * deltaSeconds;
      node.position.z += vz * deltaSeconds;

      node.position.z = Math.max(
        -maximumDepth,
        Math.min(maximumDepth, node.position.z),
      );
    }
  },
};
```

# 9. Engine 主迴圈

固定時間步可以讓物理在 Wallpaper Engine 降到 30 FPS 時仍保持近似行為。

```ts
// src/core/createEngine.ts

import type {
  PresetDefinition,
  Renderer,
} from "./contracts";
import {
  createEmptySimulationState,
} from "./state";
import type {
  SimulationState,
} from "./state";
import type {
  Viewport,
} from "./types";

export interface FoldedLatticeEngine {
  start(): void;
  stop(): void;
  resize(viewport: Viewport): void;
  getState(): Readonly<SimulationState>;
  dispose(): void;
}

export function createEngine(
  preset: PresetDefinition,
  renderer: Renderer,
  initialViewport: Viewport,
): FoldedLatticeEngine {
  const config = preset.config;
  const state =
    createEmptySimulationState(initialViewport);

  state.time.fixedDelta =
    1 / config.performance.fixedSimulationFps;

  state.topology = preset.topologyBuilder.build(
    initialViewport,
    config,
  );

  let animationFrameId = 0;
  let running = false;
  let lastTimestamp = 0;
  let accumulator = 0;

  function fixedUpdate(deltaSeconds: number): void {
    for (
      const system of preset.simulationSystems
    ) {
      system.update(
        state,
        config,
        deltaSeconds,
      );
    }
  }

  function frameUpdate(deltaSeconds: number): void {
    for (const system of preset.frameSystems) {
      system.updateFrame(
        state,
        config,
        deltaSeconds,
      );
    }
  }

  function frame(timestampMs: number): void {
    if (!running) {
      return;
    }

    if (lastTimestamp === 0) {
      lastTimestamp = timestampMs;
    }

    const rawDelta =
      (timestampMs - lastTimestamp) / 1000;

    lastTimestamp = timestampMs;

    /**
     * 回到桌面時可能突然出現幾秒 delta。
     * 直接限制在 50ms 內。
     */
    const frameDelta = Math.min(
      Math.max(rawDelta, 0),
      0.05,
    );

    state.time.delta = frameDelta;
    state.time.elapsed += frameDelta;
    state.time.frame += 1;

    accumulator += frameDelta;

    let subSteps = 0;

    while (
      accumulator >= state.time.fixedDelta &&
      subSteps <
        config.performance.maximumSubSteps
    ) {
      fixedUpdate(state.time.fixedDelta);

      accumulator -= state.time.fixedDelta;
      subSteps += 1;
    }

    /**
     * 若累積太多，不試圖一次補完，
     * 避免 spiral of death。
     */
    if (
      subSteps >=
      config.performance.maximumSubSteps
    ) {
      accumulator = 0;
    }

    frameUpdate(frameDelta);
    renderer.render(state, config);

    animationFrameId =
      requestAnimationFrame(frame);
  }

  return {
    start() {
      if (running) {
        return;
      }

      running = true;
      lastTimestamp = 0;
      animationFrameId =
        requestAnimationFrame(frame);
    },

    stop() {
      running = false;
      cancelAnimationFrame(animationFrameId);
    },

    resize(viewport) {
      state.viewport = viewport;
      renderer.resize(viewport);

      /**
       * V1 先在 resize 時重建 topology。
       * 未來可以改成保留歷史並縮放位置。
       */
      state.topology =
        preset.topologyBuilder.build(
          viewport,
          config,
        );
    },

    getState() {
      return state;
    },

    dispose() {
      running = false;
      cancelAnimationFrame(animationFrameId);
      renderer.dispose();
    },
  };
}
```

# Folded Lattice V1 規格書

## Preset 01：Breathing Membrane

---

# 0. 專案定位

## 專案名稱

**Folded Lattice**

## V1 作品名稱

**Breathing Membrane**

## 一句話描述

一張由不規則三角晶格構成的薄膜，在緩慢擾動下持續呼吸；局部壓力沿結構傳遞，留下極淡的摺痕記憶，使畫面不斷在平衡、變形與回復之間擺盪。

## V1 目標

做出一個：

* 適合 Wallpaper Engine 的 **Web 壁紙**
* 長時間觀看不疲勞
* 具有「生命感」但不明顯像某種經典演算法
* 有線、面、關係、摺疊、擴散的變化
* 可作為未來其他 preset 的共同核心

---

# 1. 設計原則

這份規格最重要的是先定「不要偏掉」的原則。

## 1.1 氣質目標

V1 應該讓人感覺：

* 安靜
* 緩慢
* 有內在秩序
* 像某種會記憶的結構
* 不是工具，不是儀表板，不是特效秀

## 1.2 避免的方向

V1 不應該像：

* 粒子特效展示
* Boids 群聚 demo
* 物理布料教學
* 音樂播放器 visualizer
* 太明顯的 cellular automata
* 工程分析圖
* 滿版閃爍、一直吸引視線的動畫

## 1.3 視覺策略

整體採：

* **80% 安靜**
* **20% 事件**

也就是大部分時間只是細微起伏，
偶爾某些邊和三角面才顯影、形成一小段摺痕。

# 2. V1 範圍

## 2.1 必做

V1 必須包含：

1. 不規則三角拓撲
2. 緩慢的膜狀變形
3. 線條依張力顯影
4. 三角面依摺疊感顯影
5. 2～4 個緩慢移動的壓力源
6. 基本記憶機制
7. 滑鼠局部擾動
8. 基本 Wallpaper Engine 參數控制

## 2.2 不做

V1 先不要做：

* 音樂反應
* 真實 3D
* 多層複雜材質
* 大量連線斷裂與重構
* 多種互動模式
* 完整 preset 切換 UI
* 複雜後處理
* 亮色、高彩風格
* 太多控制選項

---

# 3. 視覺規格

---

## 3.1 背景

### 目標

讓畫面有深度，但不要搶戲。

### 規格

* 深色背景，非純黑
* 建議色系：

  * `#0b0d10`
  * `#0e1114`
  * `#101216`
* 可加入非常微弱的亮度雜訊或霧感
* 背景不可有明顯紋理或大面積漸層運動

### 原則

背景是空氣，不是主體。

---

## 3.2 線條（Edge Layer）

### 目標

表現結構、張力與摺痕方向。

### 規格

* 線條細，建議 0.5～1.5 px 範圍內動態變化
* 預設低透明度
* 高張力時略亮、略粗
* 一般狀態下不可全部可見

### 基礎色

* 灰白
* 冷灰
* 淡青灰

例如：

* `rgba(220, 228, 232, alpha)`
* `rgba(185, 205, 210, alpha)`

### 顯示策略

* 平時只顯示部分邊
* 張力高的邊更明顯
* 舊摺痕附近邊可略微保留殘影

---

## 3.3 三角面（Triangle Layer）

### 目標

讓三角結構有「膜」與「摺面」的感覺，但不能變成實心拼圖。

### 規格

* 只使用非常淡的透明填色
* 三角面不是全時顯示
* 顯影依 `foldValue`、局部一致性與亮暗方向決定
* 可有極淡的明暗差，模擬凹/凸的空間暗示

### 原則

面應該像薄霧或薄膜，不是色塊。

---

## 3.4 高亮事件（Highlight Layer）

### 目標

在少數時刻讓局部結構被看見。

### 表現內容

* 高張力邊
* 長摺痕
* 滑鼠附近局部亮度提升
* 壓力源通過時的局部顯影

### 原則

不可太常出現，也不可大面積持續發亮。

---

# 4. 結構規格

---

## 4.1 節點分布方式

### 採用方案

**Poisson Disk Sampling + Delaunay Triangulation**

### 原因

* 分布自然
* 不規則但不混亂
* 很適合三角網
* 可延伸到其他 preset

## 4.2 節點數

建議初值：

* 小畫面：80～100
* 一般桌布：100～140
* 高解析可到 160

### V1 建議預設

**120 節點**

這會是比較穩的起點。

## 4.3 邊與三角形

由 Delaunay 自動建立。

### 要求

* 保留鄰接資訊
* 保留三角面資料
* 能快速查詢某節點相鄰邊、相鄰三角形
* 保留每條邊的 restLength
* 保留每個三角形的 fold 相關狀態

# 5. 模擬規格

---

## 5.1 模擬概念

V1 不追求物理正確，追求「膜感」。

重點是：

* 被牽連
* 有回彈
* 有遲滯
* 有記憶
* 不抖、不硬、不彈珠感

---

## 5.2 節點狀態

每個節點至少包含：

* `position`
* `velocity`
* `restPosition`
* `accumulatedForce`
* `mass`
* `neighbors`
* `memoryOffset`（可選）

---

## 5.3 邊狀態

每條邊至少包含：

* `a`, `b`
* `restLength`
* `currentLength`
* `tension`
* `memory`
* `visibility`
* `highlight`

---

## 5.4 三角形狀態

每個三角面至少包含：

* `nodes[3]`
* `center`
* `area`
* `orientation`
* `foldValue`
* `visibility`
* `phase`
* `memoryBias`

---

## 5.5 力學簡化

V1 可用以下簡化模型：

### A. 邊彈簧

每條邊對兩端節點施加回復力：

* 長於 `restLength` 時拉回
* 短於 `restLength` 時推開
* 力不需要過強

### B. 阻尼

節點速度持續衰減，避免抖動與無限震盪。

### C. 外部壓力場

壓力源影響附近節點，使其偏離平衡位置。

### D. 緩慢鬆弛

系統持續傾向回到某種柔和穩定狀態，而非完全靜止。

---

# 6. 擾動系統（Field System）

---

## 6.1 設計原則

所有動態來源都視為「field」。

V1 先實作三種：

1. `pressureField`
2. `ambientDriftField`
3. `mouseInteractionField`

---

## 6.2 Pressure Field

### 目標

提供主要生命感來源。

### 數量

預設 2～4 個。

### 每個 pressure source 屬性

* `position`
* `radius`
* `strength`
* `polarity`（凹 / 凸）
* `velocity`
* `wanderSeed`

### 行為

* 在畫面中非常緩慢移動
* 對附近節點施加柔和影響
* 不要突然跳位置
* 不要同步運動

### 效果

使局部出現：

* 凹陷
* 隆起
* 拉扯
* 緩慢傳播的摺感

---

## 6.3 Ambient Drift Field

### 目標

避免畫面只有壓力源經過時才活。

### 規格

* 一個很弱的背景流動場
* 對節點有極小偏移
* 不應造成明顯方向性
* 只是讓系統持續處於呼吸狀態

---

## 6.4 Mouse Interaction Field

### 目標

提供輕微互動，而不破壞作品安靜性。

### 行為

* 游標靠近時形成柔和局部凹陷
* 移動時可留下輕微拖曳
* 離開後緩慢回彈
* 不要產生爆炸、排斥波或明顯特效

### 可調參數

* 強度
* 作用半徑
* 回彈速度

---

# 7. 記憶系統（Memory System）

---

## 7.1 目標

讓畫面不是每秒都像重新開始。

## 7.2 V1 實作內容

只做兩種記憶：

### A. Edge Memory

高張力邊被長時間拉扯後：

* `restLength` 可產生極小偏差
* 或 `memory` 累積，使其未來更容易顯影

### B. Triangle Fold Bias

某三角面若反覆朝某方向形成摺感：

* `memoryBias` 累積
* 未來更容易再次往相同方向顯影

---

## 7.3 表現效果

觀者會感覺：

* 某些摺痕像曾經存在過
* 某些區域比較容易再被喚醒
* 整張膜像有歷史，而不是隨機動畫

---

## 7.4 強度原則

記憶不可太強。

如果太強，畫面會固定死掉。
如果太弱，則看不出存在。

V1 應該讓記憶只是一種偏向，而不是永久痕跡。

---

# 8. 顯影系統（Reveal System）

這是整個作品成不成立的核心之一。

---

## 8.1 Edge Reveal

每條邊的可見度由以下因素共同決定：

* `tension`
* 附近 `memory`
* 是否靠近 pressure source
* 是否靠近滑鼠
* 局部結構是否形成連續摺痕

### 表現

* 張力高：更亮、更清晰
* 張力低：幾乎消失
* 記憶高：較容易保留弱殘影

---

## 8.2 Triangle Reveal

每個三角面的可見度由以下因素決定：

* `foldValue`
* `memoryBias`
* 相鄰三角面的方向一致性
* 局部亮暗關係

### 表現

* 當某區形成摺面時，一小片三角面會一起顯現
* 當事件過去時，三角面逐漸退回背景

---

## 8.3 顯影密度控制

整體畫面應控制在：

* 平時只有少量結構清晰可見
* 事件發生時局部密度上升
* 事件結束後慢慢退去

### 直觀標準

如果一眼看上去整張網都很清楚，那就太吵了。

---

# 9. 渲染規格

---

## 9.1 渲染順序

建議：

1. 背景
2. 三角面淡填色

* 看起來像一件作品
* 不是單純網格 demo
* 有三角關係，但不會太硬
* 有線與面的層次
* 不會太滿、太吵

* 沒有明顯循環感

* 不會過度吸引視線

* 造成局部柔和凹陷
* 結構輕微被帶動
* 不會像遊戲特效
* 離開後緩慢回復

觀看 5 分鐘左右應該：

* 感覺某些區域有歷史
* 某些摺痕比較容易再現
* 但不會固定成一張不再變化的圖

# 方案一：〈呼吸膜〉

## 核心概念

一張懸浮在黑暗中的柔性晶格，持續進行非常緩慢的收縮與舒張。局部偶爾受到看不見的壓力，形成凹陷、隆起與短暫摺痕，之後再回到近乎平坦的狀態。

它不是在「生長」，而像在**維持某種尚未完全穩定的形體**。

## 畫面

* 節點分布大致均勻，但不形成規整網格
* 節點之間連成不規則三角網
* 平時大部分線條幾乎不可見
* 只有受到張力的邊逐漸顯現
* 三角面透過極淡的明暗差產生立體錯覺
* 偶爾出現一道長摺痕，把整張膜暫時分成兩個傾斜面

視覺上可以介於：

* 薄紙
* 皮膚
* 地形
* 液體表面
* 建築張力膜

之間，不必明確指向任何一種材質。

## 動態規則

膜內部會產生低頻的壓力源。

一個壓力源出現後：

1. 周圍節點緩慢抬起或下陷
2. 鄰近節點因牽連而被帶動
3. 影響沿著邊向外傳遞
4. 距離越遠，位移越弱、越延遲
5. 壓力消失後，膜緩慢回彈
6. 部分摺痕不會完全消失，而留下微弱的「形變記憶」

這個形變記憶很重要。它可以避免整個畫面只是循環播放的柔波。

新的壓力經過舊摺痕時，傳遞方向會被稍微改變。因此晶格會慢慢形成自己的歷史。

## 桌布體驗

這是三個方案中最安靜、最耐看的。

大部分時間只是非常細微地活動，偶爾才形成一個明顯結構。它不會持續吸引注意，但當你真的盯著看，會發現它並非重複動畫。

## 滑鼠互動

游標不必直接推開節點，而可以被視為一個有重量的物體：

* 游標停留時，膜逐漸向下凹
* 快速移動時，只留下短暫拖曳
* 離開後，凹陷緩慢回彈
* 長時間停留可能形成一道很淡的永久摺痕

互動會像「觸碰材質」，而不是粒子特效。

## 色彩

最適合單色或近似單色：

* 深灰黑底＋霧銀線
* 深藍黑底＋低飽和青灰
* 暖黑底＋很淡的米白或琥珀

立體感主要依靠亮度，不依靠彩色漸層。

## 優點

* 長時間觀看負擔最低
* 最不像演算法展示
* 很容易形成自己的氣質
* 滑鼠互動自然
* 三角網格存在，但不會一直搶戲

## 風險

如果位移太平滑，會變成普通的布料模擬；如果明暗太強，又會像 3D 技術展示。

關鍵是讓摺痕帶有一點不自然的傳播和記憶，保持抽象性。

---

# 方案二：〈應力地圖〉

## 核心概念

畫面不是展示膜本身，而是展示一張結構如何承受壓力。

你看到的線，不只是幾何邊界，而是力量通過的路徑。局部擾動會重新分配整個網絡的張力，讓一些連線增強、另一些逐漸失效。

這個版本更像一座始終在尋找平衡的建築。

## 畫面

* 節點比〈呼吸膜〉更少
* 三角形尺寸差異更明顯
* 某些區域細密，某些區域非常空
* 受力線會變亮、變粗或略微彎曲
* 未受力的線慢慢退入背景
* 三角面本身很淡，主要觀看的是力如何穿過整體

有時畫面會形成一條清楚的主力路徑，像：

* 樑柱
* 裂縫
* 閃電
* 葉脈
* 地質斷層

但它又不完全屬於任何一種。

## 動態規則

系統中會隨機出現兩類事件：

### 壓縮

幾個節點互相靠近，使周圍結構變得緊密。

壓力沿鄰邊傳遞，形成短暫的高張力區。高張力三角形會變得更明顯，也可能產生翻折。

### 拉伸

局部節點彼此遠離，部分邊被拉長。

被拉得過度的邊不一定直接斷裂，而是：

1. 先逐漸變細
2. 周圍力量重新繞路
3. 原連線消失
4. 附近節點重新建立新的連接

因此整個拓撲會慢慢改變。

這會讓畫面不是單純變形，而是連「誰與誰相連」都會重組。

## 特殊事件：結構翻面

當一個三角形承受過度壓縮時，它可以突然翻向另一側。

不是爆炸式跳動，而是一次短促、乾淨的幾何事件：

* 三角面收窄
* 接近一條直線
* 再向另一側展開
* 周圍幾個三角形被牽動

這種事件不宜頻繁。可能幾分鐘才發生一次，反而會很有存在感。

## 桌布體驗

比〈呼吸膜〉稍微更有事件性。

遠看是一張安靜的結構圖；偶爾你會看到某條力線逐漸形成，接著某一區域改變連接方式，整個構圖因此重新平衡。

它適合希望桌布有一點「事情正在發生」，但又不想要粒子亂飛的人。

## 滑鼠互動

游標可以改變局部重力方向：

* 靠近時，節點受到橫向拉力
* 拖動會改變周圍的主要受力路徑
* 快速掠過可能短暫切斷弱連線
* 停留時，結構則會逐漸向游標建立支撐

這個互動更接近「重新安排力」，不是按壓表面。

## 色彩

適合比較冷、理性的組合：

* 黑底＋冷白結構
* 石墨灰底＋淡藍應力線
* 深褐黑底＋骨白線
* 大部分低對比，只有最高張力區略亮

不同張力最好仍以亮度和透明度表達，不要使用紅黃綠工程色階，不然會像有限元素分析畫面。

## 優點

* 結構感最強
* 每次運行都會形成不同構圖
* 三角幾何具有明確存在理由
* 比生命遊戲更有整體牽連感
* 很容易產生「建築性的生命」

## 風險

若線條過多或重組太頻繁，會像網絡視覺化；若斷裂效果太戲劇化，又會像螢幕保護程式。

它需要刻意控制事件密度。

# 方案三：〈相位摺面〉

## 核心概念

畫面裡並沒有真的物理變形。

所謂的摺疊，是不同波動、方向與亮度在晶格上疊加後，產生的一種視覺錯覺。三角形有時像向前凸起，有時又像向後凹陷，但沒有穩定的正反面。

這個版本關注的不是材質，而是**觀看者如何從平面線索中感受到空間**。

## 畫面

* 晶格非常細密，但只顯露一小部分
* 幾條極慢的波穿過畫面
* 波經過時，三角形的明暗方向改變
* 相鄰三角形可能呈現矛盾的光源
* 某個區域看似摺起，但幾秒後又變成凹陷
* 部分邊界會在視覺上連成一條長摺線，之後自行解散

它會有一點：

* 光學錯覺
* 結晶
* 金屬薄膜
* 電子顯微鏡影像
* 無法成立的幾何空間

的感覺。

## 動態規則

畫面中存在幾個不同的相位場：

* 一個控制局部高度錯覺
* 一個控制亮暗方向
* 一個控制線條顯現
* 一個控制三角面翻轉感

這些場以不同速度移動。

當它們短暫對齊時，就會形成清楚的摺面；當它們錯開，結構又會瓦解成難以辨認的平面紋理。

所以畫面不是「物體動了」，而是**空間詮釋一直改變**。

## 特殊事件：方向反轉

偶爾整張畫面的明暗關係會非常緩慢地反轉：

* 原本凸起的區域逐漸變平
* 邊界模糊
* 接著看起來變成凹陷
* 但實際幾何位置可能完全沒動

這種效果會比真的把網格翻面更神祕。

## 桌布體驗

這是三個方案中最藝術、最曖昧的。

它不太像模擬任何自然物體，也不需要讓人理解系統規則。使用者可能只是偶爾發現：「剛剛那一區看起來是凸的，現在怎麼變成凹的？」

非常適合做成長時間觀看的抽象作品。

## 滑鼠互動

游標不是施加物理力量，而是改變觀看條件：

* 游標附近的虛擬光源方向改變
* 經過的三角面發生正反錯覺
* 游標停留時，附近相位逐漸同步
* 離開後，同步狀態緩慢解開

這讓互動仍然保持抽象，不會破壞作品氣氛。

## 色彩

適合非常克制的金屬、冷光感：

* 黑底＋銀灰
* 深靛底＋冰藍灰
* 暗紫黑＋灰粉
* 偶爾可以出現很輕的虹彩，但範圍不能太大

它可以比前兩者多一點顏色，因為顏色本身也能參與深度錯覺。

## 優點

* 最不容易被一眼認出使用了什麼演算法
* 視覺辨識度最高
* 不需要真實物理模擬
* 能把三角形用得很含蓄
* 有機會形成很獨特的個人作品

## 風險

如果缺少穩定的視覺支點，可能只剩下一片抽象噪聲；如果光影變化過快，又會像播放器 visualizer。

它需要保留幾個長時間存在的摺面，讓人有東西可以追蹤。

# 我最推薦的正式方向

我會以 **〈呼吸膜〉作為主體，加入少量〈應力地圖〉的結構記憶**。

也就是：

* 平時像一張安靜的柔性膜
* 局部壓力沿三角結構傳遞
* 絕大部分變形都會復原
* 少數事件留下淡淡摺痕
* 舊摺痕影響之後的波如何通過
* 很偶爾才發生一次連線重組

這樣既有材質感，也有歷史性。畫面不只是永遠重置的動畫，而會在數小時內逐漸形成自己的性格。

## 可以用的一句作品描述

> 一張持續尋找平衡的膜。
> 每次擾動都會消散，卻不會完全消失；殘留的摺痕，改變下一次力量通過的方向。

# 第一版應該先限制的範圍

先不要放入斷裂、音樂反應、天氣或複雜生成規則。

第一版只需要：

1. 一張不規則三角網
2. 兩到四個緩慢移動的壓力源
3. 節點間有柔性牽引
4. 張力高的邊才顯現
5. 三角面透過淡明暗呈現摺疊
6. 一小部分形變會留下記憶
7. 游標可以緩慢壓下膜面

做到這裡，作品是否成立其實就已經能看出來了。最關鍵的不是規則多，而是**那張膜的動作是否像在思考，而不是像在播放動畫。**



可以，這方向我反而覺得**比公車更適合桌布**。

你現在想要的感覺，我先幫你拆一下，大概是：

* 有一點 **生命遊戲／自組織** 的味道
* 但不要太像「我在看某個明確 cellular automata」
* 想要有 **線、關係、擴散、局部影響**
* 畫面應該是 **持續變化**，但不能太躁、太像特效展示
* 偏向 **抽象生成藝術**，不是資訊面板

這其實很有潛力。
如果做得好，會是一種「桌面上有個安靜生長中的系統」。

# 我先給你一個總方向

比起「康威生命遊戲變體」，我會更建議你想成：

## 「由點、線、面構成的低速關係場」

也就是不是用經典格子規則去跑，
而是讓畫面裡的元素依據一些**簡單局部規則**慢慢改變：

* 點會靠近 / 遠離
* 線會建立 / 斷裂
* 某種狀態會沿著連線傳遞
* 群體會擴散、收縮、轉相
* 局部會形成短暫秩序，再慢慢崩掉

這樣就有生命遊戲那種「看起來像有內在規則」的味道，
但不會太明顯是某個老演算法。

# 我幫你發散幾個可做的核心母題

---

## 1. 三角菌落

這個最貼近你剛提的三角組合。

### 畫面語言

* 畫面中有很多點
* 點之間若距離合適，就形成細線
* 三個互相接近的點會自動形成一個淡淡三角面
* 三角面不是永遠存在，而是像菌落一樣短暫生成、擴散、瓦解

### 動態規則

* 某些點帶有「活性」
* 活性會沿連線慢慢傳遞
* 活性高的區域更容易閉合成三角
* 三角太密時會崩解，重新鬆開
* 所以畫面會反覆在：

  * 散點
  * 線網
  * 三角群落
  * 再崩散
    之間循環

### 視覺感受

* 像微生物聚落
* 像神經突觸短暫同步
* 有一點建築結構自己長出來的感覺

### 好處

* 很適合桌布
* 不需要太花
* 「線關係與擴散」會很自然

---

## 2. 呼吸網路

這個比較偏「關係」本身。

### 畫面語言

* 一個稀疏的節點網路
* 線條細、透明、很克制
* 節點偶爾像呼吸般脈動
* 脈動會沿著相鄰節點擴散

### 動態規則

* 每個節點有自己的相位
* 相近節點會互相拉同步
* 過度同步後又逐漸失鎖
* 所以畫面會出現局部同步區塊，像波一樣傳開

### 視覺感受

* 很像一種安靜的社會網絡／神經網絡
* 比生命遊戲更柔和
* 很適合黑底桌布

### 關鍵特色

這個不是「細胞死活」，而是「關係在呼吸」。

---

## 3. 擴散裂解場

這個更有「相對擴散變化」的感覺。

### 畫面語言

* 畫面一開始只有少數種子
* 種子向外釋放影響
* 影響碰撞後形成邊界
* 邊界再被局部牽引成線、裂縫、網格

### 動態規則

* 每個種子不是單純擴散，而是有方向性
* 擴散碰撞後，會沿碰撞界面長出短線
* 短線再慢慢連成局部結構
* 結構太穩定時，又被新的擾動打散

### 視覺感受

* 像玻璃結霜、裂紋、菌絲、地形生長的混合
* 比較偏自然生成
* 「演算法味」會更淡

---

## 4. 漂浮群體 + 關係顯影

這個更輕、更桌布。

### 畫面語言

* 平常只看到很多緩慢漂浮的小點
* 當點彼此靠近，線才暫時顯現
* 當局部密度高，會出現薄薄的三角面或多邊形陰影
* 結構稍縱即逝

### 動態規則

* 所有元素都在低速漂移
* 只有接近時才「顯影」出關係
* 關係不是常駐，而是事件性的

### 視覺感受

* 很有「看見關係生成」的感覺
* 比較詩性
* 很不吵

### 適合你如果想要

* 桌面看久不煩
* 不要太數學展示
* 想保留一點曖昧性

---

## 5. 摺疊晶格

這個比較幾何、比較設計感。

### 畫面語言

* 不是自由散點，而是隱約存在一個晶格
* 晶格會局部扭曲、折疊、鬆開
* 三角形、菱形、折線在其中時隱時現

### 動態規則

* 某些節點移動會牽動鄰近結構
* 像布料、膜、摺紙那樣連動
* 局部壓力會向周圍傳遞
* 結果是幾何秩序一直在「快形成」與「快崩掉」之間

### 視覺感受

* 更建築、更理性
* 有一種冷靜的機械生命感

如果你喜歡三角，但不想太「生物」，
這個會是很好的一條路。

---

## 6. 線性生長體

如果你特別在意「線」而不是「點」。

### 畫面語言

* 一開始幾乎只有幾條細線
* 線像植物莖脈一樣慢慢長
* 分岔、試探、接上別的線
* 形成局部網絡，再逐漸枯萎回收

### 動態規則

* 每條線前端像 agent
* 會找最近的吸引點或避開擁擠區
* 兩條線接近時可能融合
* 過密則退化消失

### 視覺感受

* 比生命遊戲更 organic
* 很像「關係自己長出來」
* 很適合極簡黑底

---

# 如果你怕「太演算法」，要注意三件事

這很重要。

## 1. 不要讓規則太可辨識

一旦觀者能快速看出：

* 這是粒子系統
* 這是 boids
* 這是生命遊戲
* 這是 voronoi
* 這是 flow field

味道就會太工程展示。

### 解法

把規則混一點：

* 局部吸引 + 連線閾值
* 緩慢擴散 + 隨機衰退
* 密度驅動 + 相位同步
* 結構生成 + 結構崩解

也就是**兩三種簡單規則混合**，不要單一規則裸奔。

---

## 2. 不要讓畫面一直在「炫」

桌布最怕一直提醒你它存在。

### 解法

* 低對比
* 低速
* 少量亮點
* 變化有停頓感
* 讓一部分畫面長時間接近靜止

你可以把它想成：
**80% 安靜，20% 事件。**

---

## 3. 不要讓元素全時可見

如果所有線、點、面都一直在那裡，會變很吵。

### 解法

做「顯影邏輯」：

* 靠近才出線
* 活性高才出面
* 同步時才發光
* 衰退後逐漸透明

這會讓「關係」變成被看見的瞬間，而不是背景噪音。

# Wallpaper Engine 上很適合的特性

你可以很輕地加一些，不要太重：

  * 白天更疏、更淡

# 如果你要快速做原型，我建議先選一個公式

## 最適合第一版的原型公式：

### 「漂浮節點 + 距離連線 + 活性擴散 + 三角閉合」

具體就是：

1. 有 40～120 個慢速漂浮點
2. 距離小於某值就連線
3. 某些點有活性值

7. 全系統持續在疏散 / 凝聚之間循環

這一版就已經很夠看了。
而且：

* 有線
* 有關係
* 有相對擴散
* 有三角可能性
* 有生命感
* 又不會太像教科書演算法
