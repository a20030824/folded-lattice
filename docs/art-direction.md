# Art direction

這份文件描述 Folded Lattice 的現行藝術方向。它用來判斷一項視覺、動態或互動決策是否屬於這個系列，不取代 [`architecture.md`](./architecture.md) 的程式邊界，也不取代 [`preset-roadmap.md`](./preset-roadmap.md) 的作品規劃。

## 核心命題

Folded Lattice 不把幾何當成固定裝飾，而把它視為一種會受壓、傳遞、留下記憶並逐漸消退的材料。

作品關注的不是單一物件本身，而是：

- 關係如何短暫成立；
- 結構如何因運動或壓力被看見；
- 一次事件如何留下比事件本身更久的痕跡；
- 秩序如何在凝聚、鬆散與遺忘之間緩慢變化。

每個 preset 可以擁有不同材質與敘事，但應共享這套時間感與克制。

## 桌布優先

這些作品首先是長時間存在於桌面的背景，不是需要持續吸引注意力的展示動畫。

- 約 **80% 安靜、20% 事件**。
- 大部分時間必須允許眼睛離開作品，圖示和工作內容仍是主角。
- 事件應有發生地、傳播方向和消退過程，不應整個畫面同時表演。
- 安靜不等於完全不動；細微漂移、呼吸與殘留變化可以維持生命感。
- 截圖可以成立，但真正的作品判斷應以長時間觀看為準。

## 顯影必須有原因

線、面、節點與光不應只是隨機出現。觀者不一定需要知道公式，但應感覺到顯影與材料狀態存在因果。

優先使用：

- 運動速度或波前經過；
- 張力、壓力與局部形變；
- 結構記憶及其衰退；
- 空間連續的場，而不是逐元素 hash 閃爍；
- 人的操作所造成、且能在稍後被作品回應的痕跡。

避免讓所有結構長時間全時可見。若線、點、面一直處於相近亮度，晶格會退化成普通網格背景。

## 稀缺與視覺階級

亮度、色彩與細節必須有明確階級，最亮的元素只能屬於少數事件。

建議的閱讀順序：

1. 背景與大面積材質；
2. 極低對比的結構底噪；
3. 因運動或張力浮現的線與面；
4. 記憶痕跡；
5. 少量、短暫且有意義的事件亮點。

一般狀態不應與事件爭奪相同的亮度預算。當所有東西都在發光，就不再有事件。

## 動態語言

- 慢速，但不要只是把普通動畫減速。
- 有慣性、傳播與延遲，不突然跳格或無原因消失。
- 變化需要停頓感；局部可以長時間接近靜止。
- 週期不應太容易被看穿，避免像螢幕保護程式或固定節拍 visualizer。
- 自主事件與人的互動要有不同權限；人的操作可以更明確，但仍必須服從材料規則。
- 長期記憶必須衰退，避免作品最終被永久痕跡填滿。

## 互動原則

游標不是 UI 特效的中心，而是材料世界中的一個外力來源。

- 優先讓互動改變真正的物理或狀態，再由既有顯影路徑呈現結果。
- 避免固定跟隨游標的光圈、標記或遊戲式回饋。
- 懸停宜輕，點擊或拖曳才取得較強的影響權限。
- 回饋可以延遲：被觸碰過的位置之後更容易成為事件起點，比立即爆亮更符合系列語言。
- 關閉互動後，作品仍必須是一個完整的自主系統。

## 色彩與材質

整體以有限色盤、低飽和與材質差異建立辨識度。

- 深色作品以墨黑、深藍灰、月銀為主，亮色保留給稀少事件。
- 淺色作品以紙白、暖灰、淡墨色為主，避免純白背景與過硬黑線。
- 若使用時間色溫，可讓暖色偏向「現在／正在發生」，冷色偏向「記憶／已經發生」。
- 三角面透明度應受控制，讀起來像膜、紙或地形，而不是實心多邊形拼貼。
- 線應細、半透明、帶材質，不做廉價霓虹描邊。
- 節點不是預設必須顯示的資料點，只在材質或事件需要時出現。

## Preset 角色

### Crumpled Paper

一張安靜的紙面在漂移壓力與直接觸碰下累積摺痕。

- 重點是紙的面、皺褶方向與逐漸形成的使用痕跡。
- 互動可以更直接，但不應像按壓果凍或橡膠玩具。
- 光影應服務紙張厚度與折線，不做金屬或塑膠高光。
- 畫面可以明亮，但仍要保留暖灰層次與不均勻性。

### Wandering Ink

一個單一的墨性生命在淡紙上漫遊，路徑逐漸刻入摺疊表面。

- 核心是「一個存在」及其長時間軌跡，不擴張成粒子群或魚群模擬。
- 墨線的方向、速度與停頓比大量特效更重要。
- 痕跡可以長壽，但必須有衰退和覆寫秩序。
- 生物性應保持曖昧，不需要眼睛、肢體或明確角色動畫。

### Breathing Membrane

一張滿版的半透明三角膜在黑暗中呼吸，張力與波沿表面傳遞。

- 晶格只有在運動、張力或記憶足夠時才浮現。
- 波應像材料內部傳遞的資訊，而不是雷達圈或套在表面的 shader 特效。
- 暖色事件必須稀缺；一般自主活動以冷色或低亮度呈現。
- 滿版膜不應被讀成一個懸浮 UI 面板，邊界宜在畫面之外。

### Tide Archive

緩慢移動的潮汐穿過摺疊場，輪廓線把壓力與地形歷史存成一份持續改寫的檔案。

- 地形與等高線只是材質語言，不做座標、圖例、數值或科學儀表板。
- 線條應像被潮水暫時找到的高度，而不是固定貼在網格上的裝飾。
- 記憶形成的舊輪廓需逐漸退去，保留「檔案正在被改寫」的感覺。
- 互動可以推動地形或潮線，但不能變成可操控的沙盒工具。

## 明確避免

- 霓虹科幻 HUD、網路拓撲儀表板或資料視覺化介面；
- 粒子爆炸、全畫面脈衝與一直炫耀 GPU 的效果；
- 所有節點、邊與三角形長時間同時可見；
- 無因果的隨機閃爍、逐元素 hash 顯影；
- 固定跟隨游標的光圈或遊戲式命中效果；
- 明顯的音樂節拍閃爍與傳統 audio visualizer 語言；
- 為了展示演算法而讓規則本身壓過材質與構圖；
- 只有「漂亮網格」但沒有時間、記憶或事件層次的結果。

## 新功能與新 preset 的檢查問題

提交視覺改動前，至少回答：

1. 它在大部分時間能退到背景嗎？
2. 新增的亮度或色彩是否真的值得占用稀缺預算？
3. 顯影能否由運動、張力、壓力或記憶解釋？
4. 長時間運行後會不會逐漸填滿、變亮或失去層次？
5. 互動是否改變作品內部狀態，而不是加一層游標特效？
6. 關掉互動後，作品是否仍然成立？
7. 這個效果屬於該 preset 的材質，還是可以隨意套到任何生成藝術？
8. 移除它之後，作品的核心敘事是否真的會變弱？

若多數答案不明確，先不要把效果加入正式 preset。
