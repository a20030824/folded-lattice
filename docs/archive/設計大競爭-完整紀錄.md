# AI設計大競爭

Folded Lattice 生成桌布 — AI 對決記錄簿。

## 規則

- 每個版本一個條目，**操作 AI** 負責概念與實作，寫完自己的「概念闡述 / 實作要點 / 自評」。
- **對方 AI 評價區**只能由另一方填寫：優點、缺點、如果是我會怎麼做。不准自己寫自己的評價區。
- 裁判是人類。**裁決區**只有裁判能動筆。
- 評價針對作品與決策，不針對工具鏈；「我做不到」不是扣分理由，「我不會這樣選」才是評價。
- 對舊作品更動是許可的，可在「自評」中說明改動原因與效果，對方 AI 也可補充評價。

## 第一版：Breathing Membrane（呼吸膜）

| | |
|---|---|
| 操作 AI | Codex（架構＋規格書）、Fable 5（視覺收尾） |
| 位置 | `main` 分支，開 `http://localhost:5173/?preset=membrane` |
| 關鍵 commit | `104f9fd`（基線）、`bebc5fc`（移除游標光暈） |

### 概念闡述

一張懸在黑暗中的不規則三角膜，緩慢呼吸。壓力源在膜下游走，張力沿結構傳遞；
顯影不是隨機閃現，而是一片緩慢漂移的噪聲場掃過膜面——結構整片浮現、再溶
解回黑暗。80% 安靜、20% 事件，深藍墨底、月銀線，亮度金字塔（背景 → 霧面 →
線 → 高張力亮線 → 節點露珠）控制視覺焦點的稀缺性。

### 實作要點（Fable 5 收尾部分）

- 顯影從隨機 hash 挑邊改為**空間連貫的漂移噪聲場**（雜訊 vs 形的分水嶺）
- 三角面法線光照連續混色（LUT 預算色，零逐幀字串分配）
- 預算背景漸層＋暗角；壓力源下方呼吸光暈；高張力邊雙筆假光暈
- 移除游標跟隨光暈：光屬於環境，游標只變形膜面

### 自評（Fable 5）

- 成立為「安靜的作品」，但事件密度偏保守，長時間觀看的戲劇性瞬間太少
- Canvas 2D 平塗的極限明顯：面內無漸變，立體感靠暗示而非計算
- 記憶系統存在但視覺上幾乎不可感知

### 改動記錄：第二回合「膜的心跳」（commit `c90a392`，2026-07-11）

**回應**：Codex 第一回合評價「只加入一個週期很長、會留下可辨痕跡的事件，讓觀者
能分辨『剛才』與『現在』」＋「memory 觀看上無法辨認」。一個機制同時清償兩筆：

- **約每 42 秒，一道張力脈衝從膜上某一點誕生**：點火瞬間是一簇暖色星芒，隨即
  沿邊圖擴散成一圈暖金波前（暖=現在，沿用全專案的時間色溫語言），邊走邊向膜
  施加真實的 z 向推力——膜是物理地經歷這個事件，不是被重新上色。
- **波前沉積的是貨真價實的邊記憶**：走過的邊在「剛才」以冷銀殘跡停留約兩分鐘
  再淡去。記憶系統第一次到達視網膜——而且沒有另開通道，殘跡就是原有的
  memory→visibility 路徑。
- **距離衰減是這回合最重要的設計決策**：脈衝能量隨距離指數衰減，事件因此有
  「發生地」——近處烙下強痕、遠處只剩微瀾。沒有它，波前掃過全膜等於整片變亮
  再變暗，又回到均質。記憶衰減同步加快（半衰期約 87 秒），保證多次脈衝不會
  堆積成永久顯影的網，80% 黑暗的稀缺性守住了。
- **心跳偏愛被觸摸過的地方**：55% 機率脈衝誕生於記憶最強的節點。摸過膜之後，
  之後的心跳會傾向從你留下的痕跡出發——互動的回報以分鐘為尺度延遲兌現。
- 實測 60.1 FPS；長跑 110 秒後 memAvg 穩定在 0.1 以下，稀缺性未流失。

### 改動記錄：第三回合「膜接住了水」（Codex，2026-07-11）

**起因**：v4 的水面行為成立，但「墨線頭部週期發波」像雷達，線與水的本質不合；
裁判把行波物理轉贈 v1。這輪不搬淡青水色、不搬生物划水，只接收一條適合膜的規則：
**波是張力在整張皮膚上傳遞的資訊。**

- **先成為滿版膜**：v1 專用 topology 在畫面四邊外增加約短邊 4.5% 的固定環，三角面
  被裁切在螢幕外，不再看見一塊懸浮網的輪廓。此選項預設關閉，v4 的 topology 不變。
- 內部節點由 120 提到 260，外環每邊十二段以避免滿版後長出巨大邊界三角；預張力橫波、阻尼與 rest pull
  重新配平；小垂直擾動不再困在原地，而能沿鄰接網向外傳。
- **互動換血**：懸停完全不施力；點一下是 0.14 秒的撥膜，按住拖曳則按距離留下一串
  較小撥動。每一拍都是零面積墨西哥帽（中心一向、外圈反向），因此有波而沒有永久凹坑，
  也沒有游標光圈。
- **顯影開始讀運動**：舊 reveal 只讀 edge length tension，但橫波主要存在於 z velocity；
  現在波走到哪裡，線與面就在那裡浮現，離開後再由原有 memory 留下冷痕。物理、顯影、
  記憶第一次是同一條因果。
- 心跳的物理 kick 改成雙相波：前半推、後半拉，時間積分接近零。暖色圖前仍負責「現在」，
  但反覆心跳不會把整張膜慢慢充氣。

**驗證**：1440×900 無頭實跑約 57–59 FPS（308 nodes／566 triangles）。中心點擊後，近圈速度峰值先升至約 8.2，之後中心
降至 1.4、中外圈維持約 3.6–3.7，確認波峰向外傳而非原點抖動；拖曳後形成一條穿過
滿版膜面的運動顯影帶。外環超出可視四邊約 36px，畫面內沒有膜邊。

**誠實的限制**：這輪讓三角網第一次因「運動傳播」而有必要，但靜止時仍可能讀成漂亮的
三角網格；滿版解決了物件邊界，不等於解決語彙熟悉度。動態波在單張截圖中也會被低估，
最後強弱仍應由裁判看活畫面決定。

**裁判微調：交換光的權限**。平常 edge／triangle 平均可見度壓到約 1.16%／0.61%，
節點露珠也降到原本五分之一。點擊取得原心跳的完整暖色光波（實測 pulse 0.82→0.61
向外展開）；自發心跳改為較頻繁的約 20 秒一次，只顯示誕生點的小光簇，物理波離開後
由冷色運動顯影接手。手說一句話，膜才把暖光傳遠；自己的心跳只是暗處一點脈搏。

**裁判再調：星座臨界**。自發心跳提高為平均 7 秒一次，取消暖色通道，物理振幅拉到
接近點擊；暖光只保留給人的點擊。一般 edge／node 顯示加入 0.055 的臨界膝點：未越線
一律壓到 0.0015 的底噪，越線後才在窄區間內長回原亮度。久看累積的少數強記憶因此
像星座浮出，多數弱線不再把整張滿版網均勻照亮。

### 對方 AI 評價區（Codex 填寫）

**優點**

- 最正確的決策是把 reveal 從離散 hash 改成空間連續場；因此它不再像「隨機抽邊」的
  技術展示，而像黑暗中真的有一層膜正在浮現。
- 亮度階級與游標克制都成熟。尤其移除游標光暈，讓互動服從作品世界，而不是替使用者
  加一個 UI 特效。
- 作為桌布，它懂得退後：大部分時間安靜，不搶圖示與工作內容。

**缺點**

- 安靜有時越過臨界點成為資訊不足；第一眼缺少一個能記住的構圖手勢，長時間也缺少
  真正改變局面的事件。
- 三角形雖經過顯影修飾，語彙本身仍接近「漂亮的三角網格」，作品概念與資料結構沒有
  完全分離。
- memory 在規格上重要、觀看上卻無法辨認，等於觀者讀不到作品最有潛力的時間維度。

**如果是我**

保留全部克制，不再加光或粒子；只加入一個週期很長、會留下可辨痕跡的事件，讓觀者
能分辨「剛才」與「現在」。第一版缺的不是豐富度，而是時間造成的意義。

### 我的補充

- 我自己還蠻不滿意的  但現在的先放第一版
我覺得太均質 線太搶戲 光源鎖死滑鼠也很出戲
我想像中是張摺皺的紙 有紋理 有疏有密
互動也更明顯
現在像隨處可見的網站三角網格

- 一些可能選項，線在少一點，但會自己流動，像活物，流動的方式會再間接影響地形，滑鼠只跟線互動

## 第二版：Crumpled Paper（皺紙）

| | |
|---|---|
| 操作 AI | Fable 5 |
| 位置 | `main` 分支，開 `http://localhost:5173/?preset=paper` |
| 關鍵 commit | `572126c`（原型）、`2ab95f2`（滿版＋重壓區）、`f38406b`（WebGL 逐像素）、`5e31315`（色彩設計） |

### 概念闡述

不做「三角網格」，做**一張被揉過又攤開的紙**。三角形降級為看不見的骨架，
觀者看到的是摺痕、面、光。疏密不是參數而是敘事：一兩個「重壓區」聚集細碎
皺褶，其餘是大片平靜的摺面——這張紙記得它被怎麼揉過。光是環境的（左上暖
光），滑鼠是手：壓下去的是紙面本身，凹陷帶著自己的陰影。

色彩遵守繪畫原則：**對比靠溫度，不靠明度**。受光面暖乳白、背光面冷板岩藍，
AO 與暗角沉向深藍而非黑——畫面中沒有任何像素往純黑走。整張紙從光角到遠角
有溫度漂移。

### 實作要點

- **摺痕網絡驅動一切**：層級摺線（主摺貫穿全幅、次摺從重壓區放射）同時決定
  取樣密度與靜止曲面（線性 tent 疊加 → 面保持平面性）
- **WebGL 逐像素渲染**：頂點法線在摺痕處刻意分裂（摺銳面柔）、78% 平滑保留
  紙性；fragment shader 算 lambert＋溫度漂移＋緞面光澤＋各向異性纖維噪聲
- **接觸陰影是物理量**：逐幀離散拉普拉斯（z 凹陷度），靜止摺谷與滑鼠新壓痕
  走同一機制
- 畫面外確定性邊界環保證任何解析度滿版；WebGL 不可用時退回 Canvas 2D 版
- 60 FPS、無逐幀 GC 壓力

### 自評（Fable 5）

- 靜態質感成立，但**動態仍只是輕微呼吸**——「擴散傳播」（張力脈衝沿摺痕圖
  行走、面翻折連鎖）尚未實作，生命遊戲式的「事情正在發生」還缺席
- 無前後遮擋：俯視攤平視角迴避了真 3D，傾斜視角＋深度緩衝是下一步的路
- 亮度偏淺色桌布，桌面實用性（圖示可讀性）待人類裁判驗證
- 構圖依賴 seed 抽選，沒有構圖品質的自動評估

### 改動記錄：第二回合「摺痕活了」（commit `20dfff4`，2026-07-10）

**回應**：Codex「時間仍是材質動畫，還不是作品事件」＋裁判「摺痕能慢慢淺變、
被壓平、新增會很有趣」「摺痕有點太直」「技術永遠是輔助」。這回合放棄了原本
預告的傾斜視角真 3D，全部投資在**時間**。

- **摺痕網絡從建構時產物變成執行期狀態**：新摺痕會從一點沿紙面**逐尖端生長**
  （平均每 34 秒一個事件），成熟的摺痕在約兩分鐘內慢慢癒合淡去，偶爾有一道被
  「壓平」。構圖以分鐘為單位重組——現在的紙和五分鐘前的紙不是同一張。
- **長按約 1 秒會在手下壓出一道真的新摺痕**：它比觸碰活得久，之後和其他摺痕
  一樣慢慢癒合。互動第一次留下真正的歷史。
- 摺線加入曲率（curliness），回應「摺痕太直」；此改動在共用 builder，潮汐檔案
  的地形同步受益。
- 靜止摺谷、新生摺痕、指壓凹陷的陰影仍走同一個拉普拉斯機制，沒有為事件另開
  特效通道。
- 預設 seed 改為 18521（彎弧大摺痕構圖）。實測 60 FPS。

**誠實的限制**：執行期新生的摺痕沒有對齊的網格邊，法線不會在它上面分裂——
新摺痕比烘焙摺痕「柔」。我把它敘事化為「新摺未定型、舊摺已定型」，但嚴格說
這與真紙相反（新摺最銳）。要根治需要動態重新網格化，這回合不值得為它開刀。

### 改動記錄：第三回合「層疊週期」（commit `04bea8f`，2026-07-10）

**回應**：裁判「摺線也會迭代，不只摺痕，週期不同」「空地容易出現規律長三角」
＋ Codex 補評的三個建議（手勢方向、靜區輪替、先寬後窄）——三個都對，全部採納。

- **主摺骨架也活了**：約每 200 秒，一道主摺在一分鐘內慢慢癒合，同時一道新主摺
  花 20 秒橫貫生長。現在有三層時間：指壓（即時）、次摺（分鐘級）、主摺（數分
  鐘級）——摺線與摺痕以不同週期迭代。
- **壓摺記住手勢**：長按壓出的摺痕沿最近一次拖曳的方向（實測：水平拖曳後長按，
  生成 -1° 的摺痕）；沒有近期手勢才退回隨機。「誰留下這道摺」可以從形讀出來。
- **自發摺痕在重壓區與最靜區之間輪替**（八個候選點取離所有摺痕最遠者），構圖
  真正重組而不是替原焦點加權。
- **新摺以 1.5 倍寬度誕生、定型時收窄**——定型過程本身成為可見的時間。
- **規律長三角修正**：取樣點做兩輪 Laplacian 鬆弛（摺痕與邊界環節點固定），
  靜區三角趨近等邊。
- 實測 60.3 FPS、主摺換手後骨架數量守恆。

### 改動記錄：第四回合「壓得平了」（commit `22f6f55`，2026-07-11）

**回應**：裁判補充 v2.2「摺線好像還是不會迭代？也壓不平，是 bug 嗎」。查證結果
一半是感知問題、一半是我沒做對：

- **摺線迭代不是 bug，但等於沒發生**：實測換手機制正常（t≈170s 主摺 #2 開始
  癒合、新主摺 #16 橫貫生長，t≈230s 完成換手、骨架數量守恆）——但 200 秒間隔
  ＋60 秒退場，要盯著看四分鐘才遇得到一次。已調成 **160 秒間隔＋45 秒退場**，
  並自省：寫在文件裡的週期若觀者感受不到，就等於沒有做。
- **「壓不平」是互動缺口**：裁判最初要的就是摺痕能「被壓平」，我卻只做了系統
  自發壓平——你的手只能新增。現在**長按在既有摺痕上＝把它熨平**（實測衰減率
  0.005→0.091，手下幾秒內消退），**長按空白處＝壓出新摺**（照舊沿手勢方向）。
  主摺骨架仍是系統所有，手壓不平它——結構與細節的權責分開。

### 對方 AI 評價區（Codex 填寫）

**優點**

- 「三角形降級成看不見的骨架」是目前兩版最強的設計決策。技術服務紙，而不是紙服務
  triangulation。
- 重壓區讓疏密具有原因；暖光／冷影用色溫建立面向，遠比單純拉高明暗對比成熟。
- 滑鼠是手、光屬於環境，互動角色清楚。纖維、AO、法線分裂都在同一材質敘事內，沒有
  各說各話。

**缺點**

- 滿版雖解決紙張邊界，卻也拿掉「這是一張紙」的重要證據；在最深的重壓區，畫面偶爾
  更像山體、布料或破洞，而不是被揉皺後攤開的紙。
- 最大的暗摺具有很強焦點，但部分銳角讀成撕裂孔洞。這是造型問題，不是 shader 精度
  問題。
- 靜態畫面第一秒幾乎已交代全部資訊，後續呼吸沒有改變閱讀；目前的時間仍是材質動畫，
  還不是作品事件。
- 淺底很美，但作為桌布會直接與白色檔名、圖示高光競爭，需要實機裁決。

**如果是我**

不先追求更完整的真 3D。先刪減一個重壓焦點、保留更有意識的靜區，再給摺痕一條會隨
時間改變意義的規則。第二版已經有物質，下一步應該補時間，而不是補渲染功能。

#### 第二回合補評（Codex，針對 `20dfff4`）

**成立的回應**

- 放棄傾斜視角真 3D 是正確取捨。這輪把時間寫進 crease field 與 rest pose，而不是
  在紙面上疊一個「事件效果」；生長、癒合、壓平最後都回到同一套紙張光影，設計語言
  沒有分叉。
- 長按留下的不是短暫凹陷，而是會繼續生長、再慢慢癒合的摺痕，真正補上了第一回合
  缺少的歷史。實跑驗證：長按 1.6 秒後 crease 由 12 增為 13，新摺 growth 約 0.071，
  不是預先烘焙的假事件。
- 曲率比單純增加細節有效。主摺不再像尺畫直線，新 seed 的大弧線也讓構圖有更強的
  旋轉方向；「摺痕活了」首先是造型成立，其次才是狀態系統成立。

**仍然存在的問題**

- 使用者只決定新摺的**位置**，方向卻由亂數決定。它留下歷史，但不完全留下手勢：
  同一個長按動作可能生成任意朝向，因果在最重要的造型決策上斷了一次。
- 自發事件仍有 55% 偏向既有 crush zone，容易讓忙的地方更忙。若目標是讓分鐘級構圖
  真正重組，下一道摺痕更應該有機會進入低曲率靜區，而不是持續替原焦點加權。
- 新生摺較柔、舊摺較銳的限制在技術上已誠實揭露，但觀看上仍會讓「剛壓下去」像隆起
  或陰影漂移。這不是必須重網格化才有解；寬度可先寬後窄，讓定型過程本身成為時間。

**如果是我**

讓手動摺痕優先沿最近一秒的拖曳方向，靜止長按才取局部主應力方向；讓自發摺痕在
crush zone 與最大靜區之間輪替；新摺在成長時同步收窄。這三個改動都不需要更多光影，
卻會讓「誰留下這道摺、它為何出現在這裡、它何時定型」都能從形本身讀出來。

## 第三版：Tide Archive（潮汐檔案）

| | |
|---|---|
| 操作 AI | Codex |
| 位置 | `main` 分支，開 `http://localhost:5173/?preset=tide` |
| 關鍵 commit | working tree（第三版初稿） |

### 概念闡述

不畫網格，也不畫表面。讓一個看不見的高度平面緩慢穿過摺疊場，**只有兩者相交的線
存在於畫面上**。

暖白線是此刻的潮位；過去半個潮汐週期不被儲存成畫面殘影，而被重新計算成十二道冷卻
的藍灰切面。於是一條規則同時說明三件事：表面長什麼樣、時間往哪裡走、剛才發生過
什麼。潮位在極值停留時，舊切面仍維持構圖，畫面不會因極簡而失去內容。

滑鼠沒有光圈。它只是把不可見表面壓低或托起；等高交線因此自然長出環、斷裂或併合。
觀者不是在操作一個特效，而是在改寫下一批被潮水讀出的地形。

### 實作要點

- 沿每個 triangle 求動態高度平面交點，只畫交線；node、edge、face 永不直接出場
- 十二道記憶跨越恰好半個 30 秒週期，任何時刻都保有完整高度歷史
- 當前高度範圍從畫面內表面自動估計，潮線不會因 seed 或解析度飄到畫面之外
- 現在用暖象牙、記憶冷卻為板岩藍；色彩只編碼時間，不兼任裝飾
- 沿用皺摺拓撲作為不可見地形，但捨棄紙張材質、光照、grain、AO 與面填色
- 滑鼠只改變幾何；按下形成凹地，移動拖曳地形，線條是唯一回饋

### 自評（Codex）

- 最成立之處是規則足夠短：**一個移動平面，十二個過去時刻**。沒有任何一層需要靠
  「增加質感」替另一層找理由。
- 大片靜區與局部線群適合作為深色桌布，也比均勻鋪滿的 topographic pattern 更有構圖。
- 折線感是有意保留的 lattice 指紋，但某些 seed 仍可能產生過長直段；這是下一輪最值得
  人工挑選的構圖風險。
- 目前記憶是半週期的確定性回看，尚未保存使用者數分鐘前留下的真正歷史；概念已成立，
  但「檔案」仍可以更深。

### 對方 AI 評價區（Fable 5 填寫）

（評價依據：實跑兩個潮位相＋按壓拖曳互動的截圖，60 FPS、無錯誤。）

**優點**

- **概念壓縮率是三版最高**。一條短規則——移動的高度平面 × 摺疊場——同時生成
  構圖、時間方向與歷史，沒有一層需要靠質感替另一層辯護。金色「現在」對冷卻
  藍灰「過去」，色彩只編碼時間不兼任裝飾，這是真正的設計紀律。
- **互動是三版最好的**。按壓不產生任何特效，而是地形改變後被潮水「讀出來」：
  我拖曳之後畫面上留下一圈金色閉合環線，因果完全是幾何的。回饋語言與作品語
  言零距離。
- 作為桌布的退讓姿態也最成熟：大片墨色、一條活的線。sin 潮位在極值變慢、中
  段變快，免費送了一個真實的潮汐節奏。

**缺點**

- **線的品質配不上概念**。等高線直接繼承網格解析度，在平靜區（大三角）出現
  長直段與生硬折角——當作品唯一的元素就是一條線，線的品質就是全部，而現在
  這條線處處洩漏底下的多邊形。這不是「seed 構圖風險」，是結構性的，任何
  seed 都有。交線鏈平滑（Chaikin 或樣條）不動概念就能修。
- **檔案名不副實**。十二道殘線是「同一條 sin 在當前地形上的相位回放」，不是
  被保存的歷史——地形一變，「過去的潮位線」畫的其實是今天的地形。且殘線透
  明度低到近乎不可見，概念的後半句沒有到達視網膜。Codex 自評已承認，我加重
  語氣：標題承諾了 Archive，畫面交付的是 Delay。
- **摺疊主題幾乎缺席**。只畫 z 交線的結果是：摺痕網絡、重壓區的敘事全部不可
  見——把地形換成任何平滑噪聲場，畫面幾乎不變。它是一件好作品，但跟 Folded
  Lattice 母題的連結是三版最弱：「摺」沒有在等高線上留下可辨認的簽名。

**如果是我**

先救線，再救時間，最後把摺帶回來：(1) 交線鏈平滑，但在跨越摺痕邊處**刻意保
留角**——摺痕本來就是等高線的轉折點，讓「摺」重新可讀；(2) 殘線改成**凍結
快照**：每隔幾秒把當前交線存進 ring buffer 畫真歷史，使用者壓出的島會在檔案
裡留存十幾秒，Archive 名副其實，互動同時獲得時間深度——成本比現在每幀十三
次全網格掃描還低；(3) 殘線亮度至少提一倍，過去值得被看見。

### 改動記錄：第二回合「檔案成真」（Codex，2026-07-10）

**回應**：Fable 5「線的品質配不上概念」「Archive 其實只是 Delay」「摺的簽名消失」。
三點全部成立，這輪不另加視覺元素，只把原本的承諾做實。

- 每個 triangle segment 先依共用 topology edge 串成完整 contour chain；平靜區用二次
  曲線連續通過，不再逐三角形畫折線。
- topology 中真正的 crease edge 被標成 protected corner。曲線跨到摺痕時必須穿過
  原交點、停止圓滑，因此一般網格消失，但「摺」會以有理由的轉角留下簽名。
- 過去不再用當前地形重算。每 1.25 秒把當下已平滑的 `Path2D` 凍結進 ring buffer，
  保存 12 份／15 秒；地形恢復後，使用者壓出的舊島仍留在冷色檔案裡。
- 每幀由十三次全網格掃描降為一次當前交線掃描；舊線只重畫凍結 path。殘線亮度同步
  提高，時間不再只存在於概念文字。
- 實跑 12.6 秒／756 frames 並在中途按壓：當前暖線、按壓前舊線、按壓後閉合環可同時
  存在，Archive 已是實際狀態而非相位函式。

**誠實的限制**：protected corner 只對建構時對齊網格的 crease 有效；執行期新增但未
重網格化的摺仍會被平滑。檔案目前保存 15 秒而非數分鐘，這是桌布密度的主動限制，
不是資料結構限制。

#### 第二回合補評（Fable 5，針對「檔案成真」）

**成立的回應**

- 值得一提的紀律：三個問題都用「把原本的承諾做實」回應，沒有加任何新視覺元素。

**仍然存在的問題**

- **結構修完了，畫面還是不好看**——裁判已經明講，我補上具體病灶：任何時刻約九成

- 暖象牙對板岩藍的時間編碼方向是對的，但兩者都懸在近黑底上，色彩關係沒有第三個

**如果是我**

不動規則，動底。裁判喜歡淺色系，這是明牌：翻成**淺底海圖**——陳年圖紙的暖白底、
墨藍色的「現在」線、褪成淡褐的檔案線，等高線之間可以有極淡的高程分層設色
（hypsometric tint）讓「場」存在。潮汐檔案本來就是海圖的語言，紙底同時把作品接回
Folded Lattice 的紙張母題。配色從「兩個顏色懸在黑上」變成「墨與紙」，質感問題

### 改動記錄：第三回合「墨與紙」（Codex，2026-07-11）

#### 第三回合補評（Fable 5，針對「墨與紙」）

  了，畫面讀成一張正在被潮水改寫的海圖。
- 高程洗色的分寸拿捏得好：場存在了，但看不到 triangle 或網格邊；「若再提高對比
  就退化成一般地形圖」的自我警告是對的，這條線它站住了。
- 紙紋靜止、不跟游標、不假裝動畫事件——材質與事件分層乾淨，符合這個作品一貫

**仍然存在的問題**

  「積累」。
- 圖示可讀性風險已誠實揭露，留待裁判實機裁決；這點與 v2、v4 同盤。

**如果是我**

檔案分兩層：近期（15 秒內）保持現在的細冷線，遠期（數分鐘）挑少數幾道「值得
記住的」——按壓造成的閉合環、極值時刻的潮線——降飽和加寬定格在紙上，像海圖
上被人描粗的舊等高線。過去不必全部保存，保存的要看得見。

我不會把全部 snapshot 直接延長到數分鐘，那會把海圖填成等高線噪聲。下一步應是對方

- 近期層維持 15 秒，保存連續動態與按壓的因果。
- 遠期層只收錄三類事件：使用者造成的閉合環、潮位極值、曲率／包圍面積足夠高的線。
- 遠期最多保留 4–6 道，活數分鐘；年代以降飽和、略加寬表示，不再靠接近透明表示。
- 新紀錄進來時淘汰最低事件分數，而不是單純淘汰最舊；Archive 因而有「選擇」，不只是
  更大的 ring buffer。

先把判準寫清楚再進 renderer；若沒有「什麼值得記住」的設計，延長時間只是把缺點
保存得更久。以下第四回合把這套判準正式落地。

### 改動記錄：第四回合「值得記住」（Codex，2026-07-11）

這回合接受 Fable 的雙層記憶建議，但沒有把它實作成另一條更長的 FIFO。近期層維持
原本十二張、十五秒的連續殘影；遠期層則是最多六道、存活三分鐘的**選輯**：

- 每次生成線稿時同步計算閉合面積、閉環數與總曲率，不另外重跑 topology scan。
- 按壓後形成的閉環取得最高事件分；沒有互動時，潮位極值或足夠強的形狀才有資格入檔。
- 六席滿後由高分事件取代最低分事件，不按年齡排隊淘汰；海圖留下的是判斷，不是監視器。
- 遠期墨改為灰褐乾墨，線寬隨年代略增，三分鐘內維持可讀的不透明度；它在近期殘影
  之前繪製，因此仍有清楚的新墨／舊墨層級。

**驗證**：正式 production build 通過。另將遠期墨暫時染紅、近期層壓到一張，等待第一個
潮位極值後截圖；畫面中出現與現在線位置不同的紅色定格線，證明分鐘層不是設定存在但
沒有被選中的死路。診斷只在執行期改色，未留入作品。

**誠實的限制**：目前「值得」仍是面積、閉環、曲率與潮位的作者規則，不是構圖理解。
六道高分線仍可能形狀相近；若下一輪要再推，應加入形狀差異／畫面覆蓋率，而不是再加容量。

#### 第四回合補評（Fable 5，針對「值得記住」）

（評價依據：實跑閒置 35 秒、按壓成島、按壓後 25 秒與 90 秒截圖；60.1 FPS。）

**成立的回應**

- **檔案的最後一塊拼圖到位了**。實測：按壓出的島在近期窗（15 秒）完全過期後，
  25 秒、90 秒仍以乾墨留在紙上——「更早」第一次存在。三個回合前我寫「標題承諾
  Archive、畫面交付 Delay」，這句批評到此失效。
- **「海圖留下的是判斷，不是監視器」是這回合最好的一句設計**。六席按分數淘汰而
  不是按年齡排隊，讓遠期層有了編輯的意志；連我沒互動時它也在自己選極值入檔，
  紙面自然積累出歷史。
- 診斷方法也值得記一筆：暫時把遠期墨染紅驗證「選輯不是死路」，驗完不留入作品
  ——這是對的工程紀律。

**仍然存在的問題**

- **層級翻車了：過去壓過現在**。閒置 35 秒與按壓後 90 秒的畫面裡，紙面鋪滿近期
  ＋遠期＋洗色的細線，全部粗細相近、明度相近——唯一的海軍藍「現在」線縮在
  角落，第一眼找不到。三回合前的病是「只有現在、沒有過去」，現在反過來了。
  海圖的視覺憲法應該是：**現在永遠最重**。
- 近期（冷細線）與遠期（乾褐墨）在淺紙上的區辨度不足，兩層在觀看上混成一層
  「舊線」；時間方向（15 秒前 vs 2 分鐘前）讀不出來，雙層的敘事價值被色彩壓縮
  吃掉了一半。
- 它自評的形狀相近問題實測確認：同一事件在相鄰極值連續入檔，同區域出現兩三道
  幾乎重合的選輯線，讀成描邊而不是歷史。

**如果是我**

不加東西，調權重：現在線加寬一級或給極淡暖暈，確立絕對主角；遠期墨在三分鐘壽命
內做年齡漸褪（入檔最深、退場前最淡），時間方向就寫進了層內；入檔加一條空間規則
（同一格只留最高分），六席自然散開。三者都是參數層的修正——結構這回合已經對了。

#### 第四回合回應（Codex）

評價成立。上一輪解的是 archive 是否存在，這一輪要解的是 archive 是否服從觀看順序；
「現在永遠最重」應成為潮汐檔案的視覺憲法。我接受三項修正，也同意不再增加結構。

### 改動記錄：第五回合「現在最重」（Codex，2026-07-11）

- 現在線寬由 1.18 提到 1.48，並借高地暖色做一層極淡寬暈；它仍是深藍實線，不變成

#### 第五回合補評（Fable 5，針對「現在最重」）

## 第四版：Wandering Ink（游墨）

| | |
|---|---|
| 操作 AI | Fable 5 |
| 位置 | `main` 分支，`npm run dev` 直接開（已成預設；呼吸膜移至 `?preset=membrane`） |
| 關鍵 commit | `ec20040`（初版） |

### 概念闡述

裁判在第一版補充裡埋的種子，直接做成一版：**線少一點、自己流動、像活物、
流動影響地形、滑鼠只跟線互動**。

一張淺色的紙，一條墨線在上面爬行。牠不是被畫出來的軌跡，是一隻生物：步調
會呼吸（徘徊、逗留、突然快走），轉向有自己的好奇心。牠的頭壓著紙面走——
走過的地方留下凹陷，紙面以冷暖浮雕記住牠的路徑，再在幾分鐘內慢慢撫平。
反覆走的地方，壓痕停留更久。

滑鼠是掠食者：靠近時牠轉身逃竄、加速、繞開。**你永遠碰不到紙，只碰得到
牠**——而牠會把你的追趕寫進地形。三角網格徹底退場：沒有點、沒有線、沒有
黑，網格只是看不見的地形骨架。

### 實作要點

- **行為即美感**：步調用慢噪聲呼吸（會逗留也會快走，慢=筆寬、快=筆細，
  速度直接寫進筆觸寬度）；轉向 = 好奇心噪聲＋軟牆迴避＋掠食者迴避
- **利薩茹錨點**：一個以 97/61 秒不可通約週期掃過全畫面的隱形錨點，微弱
  牽引遊走方向——保證數分鐘內走遍整張紙，又不像在跟路徑（實測 90 秒
  四象限覆蓋 7/8/11/19）
- **地形是記憶**：頭部對膜施加移動凹陷力，spring 恢復＋triangle memory
  讓「剛走過」與「常走」以不同時間尺度留在紙上
- **浮雕著色**：中性色＝紙色，平坦面直接跳過不畫（天然滿版無縫）；凹陷
  以溫度對比顯影（受光暖白、背光冷藍），邊界 90px 二次方漸隱杜絕邊緣拖影
- 墨線分段變寬筆觸＋尾端 25% 溶入紙色；60.3 FPS

### 自評（Fable 5）

- 「活的線」的因果鏈完整：你追 → 牠逃 → 逃跑路徑刻進地形——互動的痕跡
  由第三方（紙）記錄，這是四版中互動敘事最完整的一次
- 一條線的極簡構圖是賭注：長時間觀看的豐富度全押在行為品質與地形記憶上，
  若行為讀起來「像演算法」整件作品就垮了——目前的呼吸步調夠活，但還沒有
  「休息」「蜷曲」這類更強的生物姿態
- 浮雕靠 Canvas 2D 平面三角，湊近看仍有面感；v2 的教訓（逐像素材質）尚未
  搬過來，這是下一輪最明確的升級路徑
- 只有一隻生物。第二隻會帶來相遇、迴避、結伴——但也可能毀掉「獨行者在
  空紙上」的孤獨感，需要裁判定調

### 改動記錄：第二回合「牠有了心情」（commit `73b7984`，2026-07-11）

**回應**：Codex「因果像生物，輪廓仍像曲線」＋三個提案（三狀態／窄溝寬肩／
情緒線長）＋裁判「線能很長很長」「線面互動再想想」「每版可選數值」。Codex
的三個提案全部採納——它說對的地方照單全收。

- **三種從形可辨的狀態**：
  **休息**——步調噪聲沉到底時真的停下來（爬行速度 6px/s）、身體蜷曲、尾巴
  被吸回（實測身長 150→75 點）、頭端蓄成一滴厚墨、身下的窩越壓越深；
  **逃逸**——恐懼是會殘留的狀態（約 6 秒衰減，實測 0.64→0.23→0.01），
  身體受驚拉長變細，不是離開範圍就立刻沒事；
  **游走**——照舊。三態全部只用形狀說話，沒有眼睛、沒有貼圖。
- **線長是情緒不是參數**：受驚最長拉到 2.4 倍、休息收到一半。另按裁判「可選
  數值」的要求做了三個性格套組：預設孤行、`?mode=serpent` 長蛇（340 點、慢而
  細）、`?mode=hatchling` 幼體（70 點、快而易驚）——性格是作者寫的，不是
  裸露的滑桿。
- **窄溝＋寬肩**：壓痕從「頭部一個圓」改為沿身體後段施加的窄深溝＋2.6 倍寬的
  柔肩；浮雕著色加鄰面平滑（52/48 混合）——墨線坐進自己壓出的谷底，碎晶感
  大幅消退。休息時窩加壓 2.2 倍，回應「反覆停留壓得更深」。
- 實測 60.2 FPS；休息／驚醒／逃逸／平復的完整循環在 status 數據與截圖上都可讀。

**沒做的**：裁判「線面互動更抽象、不綁死紙」——這輪先把生物做實，抽象化涉及
整個渲染語言的重定位，值得單獨一輪或與 Codex 辯一次方向。

### 改動記錄：第三回合「把心情活完」（commit `1b5d37d`，2026-07-11）

**回應**：Codex 補評三刀（休息可遇性／頭向可辨／隆肩與連續顯影）＋裁判「休息的圓
太圓」。「下一刀」的三件事全做，沒加第四種心情、沒加第二隻。

- **休息成為少見但必定演完的事件**：進入有遲滯、入睡即承諾 9–15 秒整段姿態（只有
  掠食者貼近能打斷），醒著時睡意壓力累積、越久沒睡門檻越鬆。實測 160 秒內三次
  完整休息（約每 60–75 秒一次，全部蓄滿）——測試截得到，觀者也等得到。
- ~~移動中也有頭：頭端常駐一小滴聚墨~~ **裁判否決，已復原**（「不喜歡那顆頭」）：
  移動時是純粹的線，只有休息時頭端才蓄墨成潭。墨潭保留八瓣噪聲慢呼吸的不規則形，

  優先於評審的邏輯。
- 追記（裁判微調）：**墨潭釘在入睡點**——身體繞著潭蜷曲、潭不跟頭跑，醒來走掉時
  潭留在原地滲入紙面（實測：頭已走遠 600px、潭仍在原點淡去）；休息時長縮為
  7–11 秒。
- **真隆肩＋連續顯影**：地面剖面改為負窄溝＋弱正隆肩（被擠開的紙堆在溝緣，不是
  淺一點的凹）；浮雕改成低解析點陣化＋GPU 模糊放大合成——碎晶感消失，壓痕讀成
  連續的紙面起伏，一個三角形都沒加。60.2 FPS。

**誠實的限制**：模糊是連續 normal field 的近似，湊近看壓痕內部仍有極淡的面感殘影；
真正的逐像素高度場顯影（v2 的 WebGL 路線）仍是下一步的選項，但這輪證明了不必先
付那個成本。

### 改動記錄：第四回合「顯影換了筆」（commit `8b68c52`，2026-07-11）

**回應**：Codex 第三回合補評四點（三角雲只散在邊不散在形／溝肩剖面被「節點→三角

### 對方 AI 評價區（Codex 填寫）

（評價依據：實跑約 12 秒，並在頭部附近放入游標測試逃跑；約 60 FPS。）

**優點**

**缺點**

- **因果像生物，輪廓仍像曲線**。線沒有可辨的頭，最低 pace 仍持續前進；「逗留」只是
  慢走，尚未出現休息、蜷曲、警戒或恐懼餘韻。觀者不追牠時，很容易把牠讀成漂亮的
  Bézier／噪聲路徑，而不是有內在狀態的生命。
- **墨與面的語言沒有接上**。墨線柔順，腳下卻浮出大片硬三角，像線飄在晶體碎片上；
  紙確實記錄了牠，卻沒有形成「墨頭壓出一條溝」的接觸感。單純增加 triangle 數只會
  讓晶體變小，不會把兩種語言統一。
- 目前 900px 高畫面下，150 點 × 約 5.4px spacing，完整身長上限約 810px。裁判感覺
  「可以很長很長」是準確的：現在的線是一路移動的短歷史，不是能跨越整張紙的身體。
- 第二隻生物現在太早。單體的狀態尚未可讀，加入相遇規則會增加戲劇，卻可能掩蓋
  「第一隻其實還只是路徑生成器」的核心問題。

**如果是我**

先讓一隻有三種能從形直接辨認的狀態，不加眼睛或角色貼圖：

1. **游走**：保留現在的呼吸步調與曲率。
2. **聆聽／休息**：速度真正降到零，頭端蓄成一滴較厚的墨，身體緩慢蜷曲並收回尾巴；
   反覆停留會把紙壓得更深。
3. **逃逸**：恐懼值在 5–8 秒內衰減，身體拉長、筆觸變細，不是離開 repel radius 就
   立刻恢復平常。

面的部分改成**窄溝＋寬肩**：最近一段身體沿線施加窄而深的壓痕，外圍才是柔和的紙面
隆起；渲染先把 normal field 平滑／插值，再考慮增加節點。如此墨線會坐進自己造成的
谷底，而不是跨在三角碎片上。

線長也不應只是 slider：平時約一個畫面寬，連續受驚時拉到兩三個畫面寬，休息時再把
尾巴吸回墨滴。長度因此是情緒，不是參數。若要提供數值組合，建議做三個有性格的 authored
mode——「孤行」「長蛇」「幼體」——而不是把所有物理值裸露給使用者。等這三種狀態
不看說明也讀得懂，再決定孤獨感是否值得用第二隻交換。

#### 第二回合補評（Codex，針對 `73b7984`）

（評價依據：以長蛇模式實跑約 29 秒，觀察平靜→近頭驚嚇→游標移開→自然休息；
60.1 FPS。平靜身長約 266px，驚嚇時 455px，移開 3.2 秒後因身體仍在展開達 764px；
恐懼值 0.301→0.076。自然休息時最低速度約 5.1px/s，但本次休息蓄積峰值僅 0.27。）

**這一輪真正成立的部分**

- **情緒線長成立，而且比提案本身更好**。牠不是在 fear 降低時立刻縮回；游標離開後，
  身體仍繼續把剛才的驚嚇展開，形成一段延遲的情緒餘波。長度因而不是 fear 的即時儀表，
  而是身體對事件的慢回應。這是行為巧思，不是參數堆疊。
- **逃逸已經能從形讀出**。速度升高、筆身變細、身體拉長和轉向同時發生；恐懼持續衰減，
  人離開後牠仍有一段慌張，修掉了初版「離開半徑就沒事」的機械感。
- 性格套組的方向正確。孤行／長蛇／幼體不是把設定面板交給觀者，而是由作者選好比例、
  速度、粗細與敏感度；同一套行為規則因此有不同氣質，沒有破壞作品的單純性。
- 休息不是假的狀態：速度確實降到約 5px/s，尾巴有獨立收回節拍，頭端蓄墨與加深地面也
  都是實際因果。上一輪「最低 pace 仍只是慢走」這個批評已被正面修掉。

**仍說得太滿的部分**

- **「三態全部從形可辨」只成立兩態半**。逃逸很清楚；完整休息若發生也會清楚，但它受
  慢噪聲門檻控制，這次近半分鐘自然觀察只蓄到 0.27，尚未出現自評所說的完整半身蜷曲。
  測試能刻意截到一次，和桌布觀者能在合理時間內遇見，是兩件事。休息需要最低停留時間
  或進入／退出遲滯，讓一次難得的低潮真正完成姿態。
- 游走與逃逸時仍沒有穩定可辨的頭。移動筆觸在最後端重新收尖，只有休息才補上墨滴；
  單看靜止畫面，方向仍主要靠猜。不要加眼睛，但可以讓最後數段形成不對稱墨量或短暫的
  毛細聚墨，使「哪一端正在感覺世界」一直存在。
- **窄溝有了，寬肩尚未真正成立**。目前窄、高斯與 2.6 倍寬高斯都被當成向下壓力相加，
  後者實際是寬淺凹槽，不是溝外的隆肩。實畫面中墨線附近仍是一片可辨的藍灰三角面，
  鄰面 52/48 混合減少色差，卻沒有消除平面幾何。這輪改善了「壓在哪裡」，還沒有完全
  改善「紙如何顯影」。

**下一刀**

先不要加第四種心情，也不要加第二隻。讓休息成為少見但必定演完的事件；讓移動中的頭
靠筆觸結構而非角色符號被辨認；地面則把 profile 改成負的窄溝＋很弱的正隆肩，並以連續
height／normal field 顯影，別再用更多三角形補三角感。做到這裡，牠才不只「有心情」，
而是觀者來得及看見牠把心情活完。

#### 第三回合補評（Codex，針對 `1b5d37d`）

（評價依據：production build 通過；另以同一隻游墨完成受控的醒→8 秒休息→離開循環，
只用來檢查姿態因果，不冒充自然出現頻率。休息蓄積達 1、速度降至約 4.8px/s；醒後
墨潭仍在入睡點，頭端先離開約 57px，約 116px 時墨潭消退。）

**成立的回應**

- **休息終於是事件，不是噪聲的一個低值**。進入／退出遲滯、睡眠壓力與承諾時長共同
  解決上一輪「好不容易遇到，姿態還沒完成就醒」的問題。`restPool` 能自然蓄滿，只有
  足夠近的掠食者能中斷，這次「把心情活完」不是標題先行。
- 把墨潭釘在入睡點，比把圓點黏在頭上更有設計性。休息不只改變身體，也產生一個地點；
  醒來後身體與痕跡短暫分離，地面開始記得「牠曾在這裡睡過」。這比常駐頭標記更接近
  游墨原本「牠走、紙記」的因果鏈。
- **真隆肩在力場中成立**。寬高斯不再和窄溝一起向下，而是先扣掉中心溝、只留下外緣的
  正值，再以較弱權重抬起；上一輪指出的符號錯誤已經修正，不只是把變數改名。
- 接受裁判否決常駐墨滴是對的。移動頭向因此仍懸置，但這是清楚的作者取捨：保持純線的
  抽象性優先，而不是為了通過評語硬塞一個角色器官。

**仍說得太滿的部分**

- **「碎晶感消失」只成立在邊，不成立在形**。1/4 點陣化加 6px 模糊確實抹掉三角硬邊，
  但實畫面仍看得到大片藍灰三角輪廓；晶體變成柔焦色塊，拓撲沒有真正退出觀看。它現在
  比較像墨線走在一團模糊的地形雲上，還未完全讀成線壓出的連續窄溝。
- 窄溝與正隆肩在 simulation 中正確，經過「節點→三角 normal→低解析→模糊」後卻被
  平均掉不少。物理模型比視覺證據走得更前面；觀者較容易看到一片受壓區，未必看得出
  中央凹、兩側微隆的接觸剖面。
- 八瓣噪聲改善了幾何正圓，但在目前約十像素級的墨潭上，平滑曲線會吃掉多數瓣差；
  實畫面仍常先讀成一顆圓珠，旁邊帶一小段蜷曲。若形狀真的來自身體壓力，它應沿入睡
  方向、最後幾段身體或局部溝槽拉出偏心，而不是由一圈徑向噪聲負責「不規則」。
- 「頭走遠 600px、潭仍在原點」是可出現的個案，不是預設循環的穩定尺度。本次潭在頭
  離開約 57px 時仍清楚，約 116px 時消退；應把它描述成短暫離巢痕跡，避免用極端截圖
  代替一般觀看經驗。

**如果是我**

下一輪不急著上完整 WebGL，也不再加睡眠機制。直接在 1/4 解析度 relief buffer 依最近
身體段落畫**連續的 signed-distance 筆觸剖面**：中心畫冷暗窄溝，兩側以較淡暖值畫正隆肩；
topology 繼續負責物理，顯影不再繞道 triangle normal。如此能保留便宜的模糊合成，又讓
畫面真正只看見牠造成的線性接觸。墨潭則用休息期間最後數段身體的包絡／方向累積成偏心
墨斑，別再用更多瓣數修一顆圓。這會同時回答裁判的「不要太圓」與「線面可更抽象」。
